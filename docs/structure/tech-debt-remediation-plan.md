## 支撑与治理
- **质量门槛**：所有任务需更新或新增测试；若测试难以实施，需在 PR 中给出风险说明与补救计划。
- **沟通节奏**：每日同步推进状态，阻塞事项在当日提出；周报包含完成卡片、进行中卡片、风险与需求。
- **风险缓解**：
  - DOM 选择器变动频繁：优先引入测试 fixture；必要时考虑引入 Playwright 快照以防 UI 变更。
  - Schema 重构牵涉表单：先灰度到 Beta 分支，通过内测确认后再主干发布。
  - REST 调用依赖不同部署：准备可配置的 mock server，降低本地调试门槛。
