# 📝 格式优化说明 - v0.2.1

## 优化内容

根据用户反馈，对选中内容剪藏的格式进行了优化，使其更加简洁和易用。

---

## 🎯 主要变化

### 1. YAML Frontmatter 简化

#### 优化前
```yaml
---
type: clipper
title: "页面标题 - 内容预览... @142345"
page_title: "页面标题"
url: "https://example.com/article"
fragment_url: "https://example.com/article#:~:text=..."
clipped_at: "2025-10-02T14:23:45.000Z"
tags: [clipping, fragment]
---
```

#### 优化后
```yaml
---
type: clipper
title: "页面标题"
url: "https://example.com/article#:~:text=..."
clipped_at: "2025-10-02 14:23:45"
tags: [clipping, fragment]
---
```

**改进点**：
- ✅ 删除了 `page_title`（与 `title` 重复）
- ✅ 删除了 `fragment_url`（合并到 `url`）
- ✅ `title` 只显示页面标题（不包含预览和时间戳）
- ✅ `url` 直接是 Text Fragment 链接（精确定位）
- ✅ `clipped_at` 改为易读格式（YYYY-MM-DD HH:mm:ss）

### 2. 时间格式优化

#### 优化前
```
clipped_at: "2025-10-02T14:23:45.000Z"
```

#### 优化后
```
clipped_at: "2025-10-02 14:23:45"
```

**改进点**：
- ✅ 使用 24 小时制
- ✅ 去除 ISO 格式的复杂性
- ✅ 更易读，更符合中文习惯
- ✅ 精确到秒，足够唯一

### 3. 文件名优化

#### 优化前
```
页面标题 - 内容预览前30字... @142345.md
```

#### 优化后
```
页面标题 2025-10-02 14:23:45.md
```

**改进点**：
- ✅ 更简洁，不包含内容预览
- ✅ 使用完整时间戳（易读）
- ✅ 避免文件名过长
- ✅ 易于排序和管理

## 💡 设计理念

### 1. 简洁优先

**原则**：只保留必要信息，删除冗余字段

**实践**：
- 合并重复字段（`title` 和 `page_title`）
- 合并功能相同的字段（`url` 和 `fragment_url`）
- 使用更简洁的时间格式

### 2. 易用为主

**原则**：让用户能快速理解和使用

**实践**：
- `url` 字段直接可用（点击即可跳转）
- 时间格式易读（不需要转换）
- 文件名简洁明了

### 3. 避免混淆

**原则**：字段名称和内容要一致

**实践**：
- `title` 就是标题，不包含其他内容
- `url` 就是链接，不需要区分类型
- `clipped_at` 就是时间，格式统一

## 🔧 技术实现

### 时间格式化函数

```typescript
function formatDateTime(date: Date): string {
  const yyyy = date.getFullYear();
  const mm = String(date.getMonth() + 1).padStart(2, '0');
  const dd = String(date.getDate()).padStart(2, '0');
  const hh = String(date.getHours()).padStart(2, '0');
  const min = String(date.getMinutes()).padStart(2, '0');
  const ss = String(date.getSeconds()).padStart(2, '0');
  
  return `${yyyy}-${mm}-${dd} ${hh}:${min}:${ss}`;
}
```

### 文件名生成函数

```typescript
function generateClipperTitle(pageTitle: string, now: Date): string {
  const yyyy = now.getFullYear();
  const mm = String(now.getMonth() + 1).padStart(2, '0');
  const dd = String(now.getDate()).padStart(2, '0');
  const hh = String(now.getHours()).padStart(2, '0');
  const min = String(now.getMinutes()).padStart(2, '0');
  const ss = String(now.getSeconds()).padStart(2, '0');
  
  return `${pageTitle} ${yyyy}-${mm}-${dd} ${hh}:${min}:${ss}`;
}
```

## 📚 使用指南

### 查看精确定位链接

在 Obsidian 中：
1. 打开剪藏文件
2. 查看 frontmatter 中的 `url` 字段
3. Cmd/Ctrl + 点击链接
4. 浏览器会打开原网页并高亮显示内容

### 使用 Dataview 查询

**查询所有剪藏**：
```dataview
TABLE 
  title as "标题",
  clipped_at as "时间"
FROM ""
WHERE type = "clipper"
SORT clipped_at DESC
```

**查询今天的剪藏**：
```dataview
LIST
FROM ""
WHERE type = "clipper" 
  AND contains(clipped_at, "2025-10-02")
```

## 🔄 迁移说明

### 对现有剪藏的影响

- ✅ **不影响**：已有的剪藏文件不会改变
- ✅ **向前兼容**：新格式可以与旧格式共存
- ✅ **无需迁移**：不需要手动更新旧文件

### 新旧格式共存

Obsidian 中可以同时存在两种格式的文件：

```
Clippings/
├── 旧格式/
│   └── Article - Preview... @142345.md
└── 新格式/
    └── Article 2025-10-02 14:23:45.md
```

两种格式都可以正常使用，Dataview 查询也都支持。
