# 🧪 测试选中内容剪藏功能

## 快速测试步骤

### 1. 重新加载扩展

1. 打开 Chrome 扩展管理页面：`chrome://extensions/`
2. 找到 "All in Ob"
3. 点击 **刷新** 按钮（🔄）

### 2. 准备测试环境

确保：

- ✅ Obsidian 已打开
- ✅ Local REST API 插件已启动
- ✅ 扩展配置正确（在设置页面测试连接）

### 3. 测试选中内容剪藏

#### 测试用例 1：基本剪藏（带评论）

1. 打开任意网页（例如：https://www.wikipedia.org）
2. 选中一段文本（建议 1-2 个段落）
3. 右键点击选中的文本
4. 选择 **"✂️ 剪藏选中内容到 Obsidian"**
5. 在弹出的对话框中：
   - 确认预览内容正确
   - 在评论框中输入：`这是一个测试评论 💡`
   - 点击 **"✂️ 剪藏"** 按钮
6. 等待通知显示 "已保存到 Obsidian"
7. 在 Obsidian 中检查生成的文件

**预期结果**：

- ✅ 对话框正常显示
- ✅ 预览内容正确
- ✅ 文件成功保存到 Obsidian
- ✅ 文件包含评论内容
- ✅ 文件包含精确定位链接
- ✅ 文件名包含时间戳（例如：`wikipedia-selected-text-@142345.md`）

#### 测试用例 2：不添加评论

1. 选中另一段文本
2. 右键 → **"✂️ 剪藏选中内容到 Obsidian"**
3. 在对话框中**不输入任何评论**
4. 直接点击 **"✂️ 剪藏"**

**预期结果**：

- ✅ 文件正常保存
- ✅ 没有 "我的评论" 部分

#### 测试用例 3：取消剪藏

1. 选中文本
2. 右键 → **"✂️ 剪藏选中内容到 Obsidian"**
3. 点击 **"取消"** 按钮

**预期结果**：

- ✅ 对话框关闭
- ✅ 没有文件被创建
- ✅ 没有通知显示

#### 测试用例 4：ESC 键取消

1. 选中文本
2. 右键 → **"✂️ 剪藏选中内容到 Obsidian"**
3. 按 **ESC** 键

**预期结果**：

- ✅ 对话框关闭
- ✅ 没有文件被创建

#### 测试用例 5：点击外部取消

1. 选中文本
2. 右键 → **"✂️ 剪藏选中内容到 Obsidian"**
3. 点击对话框外的灰色区域

**预期结果**：

- ✅ 对话框关闭
- ✅ 没有文件被创建

### 4. 测试精确定位链接

1. 在 Obsidian 中打开刚才剪藏的文件
2. 找到 "来源" 信息框中的 **"精确定位"** 链接
3. 点击该链接

**预期结果**：

- ✅ 浏览器打开原网页
- ✅ 页面自动滚动到剪藏的内容位置
- ✅ 内容被高亮显示（Chrome/Edge）

**注意**：Text Fragment 功能在某些网站可能不工作（取决于网站的 CSP 策略）

### 5. 测试同一页面多次剪藏（重要！）

这是测试文件名唯一性的关键测试。

1. 在同一个网页上选中**第一段文本**
2. 右键 → **"✂️ 剪藏选中内容到 Obsidian"**
3. 添加评论：`第一次剪藏`
4. 点击剪藏，记录文件路径
5. 等待 2-3 秒
6. 在同一页面选中**第二段不同的文本**
7. 右键 → **"✂️ 剪藏选中内容到 Obsidian"**
8. 添加评论：`第二次剪藏`
9. 点击剪藏，记录文件路径
10. 在 Obsidian 中检查两个文件

**预期结果**：

- ✅ 生成了两个不同的文件
- ✅ 文件名包含不同的时间戳
- ✅ 第一个文件没有被覆盖
- ✅ 两个文件都包含各自的评论和内容
- ✅ 文件路径类似：
  ```
  Clippings/wikipedia.org/2025/2025-10-02/Wikipedia 2025-10-02T14:23:45.md
  Clippings/wikipedia.org/2025/2025-10-02/Wikipedia 2025-10-02T14:25:30.md
  ```

### 6. 测试完整页面剪藏（确保原功能正常）

1. 在网页空白处右键
2. 选择 **"剪藏整个页面到 Obsidian"**

**预期结果**：

- ✅ 整个页面被剪藏
- ✅ 类型为 `article` 或 `ai_chat`
- ✅ 没有 "我的评论" 部分
- ✅ 文件名不包含时间戳（使用标题 slug）

## 检查生成的文件

### 文件结构检查清单

打开生成的 clipper 文件，检查：

- [ ] Frontmatter 包含 `type: clipper`
- [ ] Frontmatter 包含 `title`（页面原始标题）
- [ ] Frontmatter 包含 `url`（Text Fragment 链接，包含 `#:~:text=`）
- [ ] Frontmatter 包含 `clipped_at`（格式：YYYY-MM-DDTHH:mm:ss）
- [ ] Frontmatter 包含 `tags: [clipping]`
- [ ] Frontmatter **没有** `page_title` 和 `fragment_url`（已简化）
- [ ] 正文直接是剪藏内容，没有额外的一级标题
- [ ] 没有冗余的"来源"信息框
- [ ] 如果添加了评论，在内容下方有 "💭 我的评论" 部分
- [ ] 评论和内容之间有分隔线（`---`）
- [ ] 内容格式正确（Markdown）
- [ ] 图片链接是绝对 URL
- [ ] 文件名格式：`{页面标题} {YYYY-MM-DDTHH:mm:ss}.md`

### 示例文件内容

**文件名**：

```
Wikipedia 2025-10-02T14:23:45.md
```

**文件内容**：

```markdown
---
type: clipper
title: 'Wikipedia'
url: 'https://www.wikipedia.org/#:~:text=Wikipedia%20is%20a%20free%20online%20encyclopedia'
clipped_at: '2025-10-02T14:23:45'
tags: [clipping]
---

Wikipedia is a free online encyclopedia...

---

## 💭 我的评论

这是一个测试评论 💡
```

**文件路径示例**：

```
Clippings/wikipedia.org/2025/2025-10-02/Wikipedia 2025-10-02T14:23:45.md
```

**格式说明**：

- ✅ YAML 只包含必要字段：type, title, url, clipped_at, tags
- ✅ url 字段包含 Text Fragment 链接（精确定位）
- ✅ title 是页面的原始标题
- ✅ clipped_at 使用易读的格式：YYYY-MM-DDTHH:mm:ss
- ✅ 文件名格式：{页面标题} {时间}.md
- ✅ 正文直接是剪藏内容，无冗余标题
- ✅ 评论在内容下方，用分隔线隔开

## 常见问题排查

### 问题：右键菜单没有显示 "剪藏选中内容"

**检查**：

1. 是否选中了文本？
2. 扩展是否已重新加载？
3. 打开控制台查看是否有错误

**解决**：

```bash
# 重新构建
cd AiiinOB/your-extension
npm run build
```

### 问题：对话框样式异常

**可能原因**：

- 网页的 CSS 影响了对话框样式
- z-index 不够高

**检查**：

- 打开浏览器开发者工具
- 检查对话框元素的样式

### 问题：内容保存失败

**检查**：

1. 打开浏览器控制台（F12）
2. 查看 Console 标签中的错误信息
3. 检查 Network 标签中的请求

**常见错误**：

- `Failed to fetch`: REST API 未运行
- `401 Unauthorized`: API Key 错误
- `Network error`: 网络连接问题
