# 选中内容剪藏功能 - 更新日志

## 版本 0.2.0 - 2025-10-02

### ✨ 新功能

#### 1. 选中内容剪藏 (Clipper)

全新的碎片化剪藏功能，让你可以快速收集网页中的精彩片段。

**核心特性**：
- ✂️ **选中剪藏**：右键选中的文本即可剪藏
- 💭 **添加评论**：剪藏前可以添加个人思考
- 🎯 **精确定位**：生成 Text Fragment URL，点击可直接跳转到原文位置
- 📝 **简洁格式**：优化的 Markdown 格式，去除冗余信息
- 🔢 **唯一文件名**：使用时间戳确保同一页面多次剪藏不会覆盖

#### 2. 优雅的对话框界面

**设计亮点**：
- 🎨 现代化的深色主题
- 👀 选中内容预览
- ⌨️ 支持 ESC 键取消
- 🖱️ 点击外部关闭
- 📱 响应式设计

#### 3. 智能文件命名

**命名策略**：
```
格式：{页面标题} - {内容预览}... @{时间戳}
示例：React Documentation - useState is a Hook... @142345
```

**文件路径**：
```
默认：Clippings/{domain}/{yyyy}/{yyyy}-{mm}-{dd}/{slug}.md
示例：Clippings/react.dev/2025/2025-10-02/react-documentation-usestate-@142345.md
```

### 🎯 使用方法

1. **选中文本**：在网页上选中你想要剪藏的内容
2. **右键菜单**：选择 "✂️ 剪藏选中内容到 Obsidian"
3. **添加评论**：（可选）在弹出的对话框中添加你的想法
4. **确认剪藏**：点击 "✂️ 剪藏" 按钮
5. **查看结果**：在 Obsidian 中查看生成的文件

### 📄 输出格式

#### 简洁的 Markdown 格式

```markdown
---
type: clipper
title: "页面标题 - 内容预览... @142345"
page_title: "页面标题"
url: "https://example.com/article"
fragment_url: "https://example.com/article#:~:text=..."
clipped_at: "2025-10-02T14:23:45.000Z"
tags: [clipping, fragment]
---

这里是剪藏的内容...

---

## 💭 我的评论

这里是你的评论...
```

#### 格式优化

- ✅ **去除冗余**：删除了重复的"来源"信息框
- ✅ **内容优先**：正文直接展示剪藏内容，无额外标题
- ✅ **评论在下**：个人思考放在内容下方
- ✅ **元数据完整**：所有链接和信息都在 YAML frontmatter 中

2. **src/background/index.ts**

   - 处理选中内容剪藏的触发

### 🆚 与其他功能的区别

| 功能 | 文件命名 | 是否覆盖 | 适用场景 |
|------|---------|---------|---------|
| **完整文章** | 基于标题 | ✅ 会覆盖 | 保存整篇文章 |
| **选中内容** | 标题+预览+时间戳 | ❌ 不覆盖 | 碎片化收集 |
| **AI 对话** | 日期+标题 | ✅ 会覆盖 | 对话记录 |

### 🔍 Text Fragment URL

使用 Web 标准的 [Text Fragments](https://web.dev/text-fragments/) 技术：

**URL 格式**：
```
https://example.com/article#:~:text=选中的文本内容
```

**浏览器支持**：
- ✅ Chrome/Edge 80+
- ✅ Safari 16.1+
- ⚠️ Firefox（需要扩展）

**功能**：
- 自动滚动到指定位置
- 高亮显示选中的文本
- 精确定位原文内容

### 📝 最佳实践

#### 1. 选择合适的内容长度
- ✅ 推荐：1-3 个段落

### 🐛 已知限制

1. **Text Fragment 兼容性**
   - 某些网站可能不支持（CSP 策略限制）
   - 动态加载的内容可能无法定位

2. **文件名长度**
   - 限制在 180 个字符以内
   - 超长会自动截断

3. **同一秒剪藏**
   - 理论上可能冲突（极少见）
   - 建议间隔 1 秒以上
