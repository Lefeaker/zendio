# 片段剪藏 - 脚注格式和上下文捕捉功能说明

## 📅 更新日期
2025-10-03

## 🎯 功能概述

为片段剪藏功能添加了两个重要的新特性：

1. **脚注格式支持** - 兼容 Obsidian Sidebar Highlights 插件
2. **上下文捕捉** - 可选择捕捉选中文字周围的上下文，并高亮标记实际选中的部分

## ✨ 新增功能

### 1. 脚注格式（Footnote Format）

**功能说明：**
- 将评论以 Obsidian 脚注格式保存
- 完全兼容 Sidebar Highlights 插件
- 评论会显示为脚注引用，点击可跳转

**格式对比：**

**原格式：**
```markdown
---
type: clipper
title: "页面标题"
url: "https://example.com"
clipped_at: "2025-10-03 14:30:00"
tags: [clipping, fragment]
---

这是选中的文本内容

---

## 💭 我的评论

这是我的评论内容
```

**脚注格式：**
```markdown
---
type: clipper
title: "页面标题"
url: "https://example.com"
clipped_at: "2025-10-03 14:30:00"
tags: [clipping, fragment]
---

这是选中的文本内容[^1]

[^1]: 这是我的评论内容
```

**优势：**
- ✅ 更简洁，没有分隔线和标题
- ✅ 兼容 Sidebar Highlights 插件
- ✅ 评论紧跟内容，阅读体验更好
- ✅ 支持多行评论（自动缩进）

### 2. 上下文捕捉（Context Capture）

**功能说明：**
- 可选择是否捕捉选中文字周围的上下文
- 用 `==高亮==` 标记实际选中的部分
- 在 Obsidian 中会显示为黄色高亮

**示例：**

**不捕捉上下文（默认）：**
```markdown
这是选中的文本内容
```

**捕捉上下文：**
```markdown
前面的上下文内容，提供了背景信息。 ==这是选中的文本内容== 后面的上下文内容，提供了补充说明。
```

**在 Obsidian 中的显示效果：**
- 前面的上下文内容，提供了背景信息。 <mark>这是选中的文本内容</mark> 后面的上下文内容，提供了补充说明。

**配置选项：**
- **上下文长度**：可设置 50-1000 字符（默认 200）
- **上下文模式**：
  - **按字符数**：精确控制上下文长度
  - **按句子数**：按完整句子捕捉上下文

## ⚙️ 配置说明

### 在设置页面配置

打开扩展设置页面，找到"片段剪藏配置"区块：

#### 1. 使用脚注格式（推荐）

- **默认值：** 启用
- **说明：** 评论将以 Obsidian 脚注格式保存
- **推荐：** 启用，以获得更好的兼容性

#### 2. 捕捉上下文

- **默认值：** 禁用
- **说明：** 捕捉选中文字周围的上下文
- **适用场景：**
  - 需要保留上下文信息
  - 选中的文字较短，需要更多背景
  - 想要在 Obsidian 中看到高亮效果

#### 3. 上下文长度

- **默认值：** 200 字符
- **范围：** 50-1000 字符
- **说明：** 前后各捕捉的字符数或句子数
- **建议：**
  - 短文本：100-200 字符
  - 中等文本：200-400 字符
  - 长文本：400-1000 字符

#### 4. 上下文模式

- **默认值：** 按字符数
- **选项：**
  - **按字符数**：精确控制，适合大多数情况
  - **按句子数**：保持句子完整性，适合文章段落

### 核心算法

#### 上下文提取算法

```typescript
function extractContextFromPage(
  doc: Document,
  selectedText: string,
  config: FragmentClipperConfig
): { before: string; selected: string; after: string } | null {
  // 1. 在页面中查找选中的文本
  const bodyText = doc.body.textContent || '';
  const selectedStart = bodyText.indexOf(selectedText);
  
  // 2. 根据模式提取上下文
  if (config.contextMode === 'chars') {
    // 按字符数提取
    beforeText = bodyText.substring(
      Math.max(0, selectedStart - contextLen),
      selectedStart
    );

      selectedEnd,
      Math.min(bodyText.length, selectedEnd + contextLen)
    );
  } else {

  return { before, selected, after };
}
```

## 🎯 使用建议

### 何时使用脚注格式？

**推荐使用：**
- ✅ 使用 Sidebar Highlights 插件
- ✅ 希望评论紧跟内容
- ✅ 追求简洁的 Markdown 格式

**可选不使用：**
- 习惯原有的分隔线格式
- 不使用 Sidebar Highlights 插件

### 何时捕捉上下文？

**推荐捕捉：**
- ✅ 选中的文字较短（少于 50 字）
- ✅ 需要保留背景信息
- ✅ 想要在 Obsidian 中看到高亮效果
- ✅ 阅读笔记需要更多上下文

**可选不捕捉：**
- 选中的文字已经很完整
- 不需要额外的上下文
- 追求最简洁的输出

### 上下文长度建议

| 场景 | 建议长度 | 说明 |
|------|---------|------|
| 短句摘录 | 100-200 字符 | 保留基本上下文 |
| 段落摘录 | 200-400 字符 | 保留完整语境 |
| 长文摘录 | 400-1000 字符 | 保留详细背景 |
| 技术文档 | 按句子数 | 保持句子完整性 |

## 🐛 已知限制

1. **上下文查找**
   - 依赖 `textContent`，可能丢失格式
   - 如果页面有重复文本，可能定位不准

2. **句子分割**
   - 简单的正则分割，可能不完美
   - 对于特殊标点可能处理不当

3. **多行评论**
   - 脚注格式需要手动缩进
   - 复杂格式可能显示不完美
