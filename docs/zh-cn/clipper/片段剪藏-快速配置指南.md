# 片段剪藏 - 快速配置指南

## 🚀 快速开始

### 1. 打开设置页面

1. 点击浏览器工具栏的扩展图标
2. 右键 → "选项" 或 "设置"
3. 或者访问 `chrome://extensions/` → 找到 "All in Ob" → 点击"详细信息" → "扩展程序选项"

### 2. 找到"片段剪藏配置"区块

向下滚动到"片段剪藏配置"区块（✂️ 图标）

### 3. 配置选项

#### 选项 1：使用脚注格式（推荐）✅

**默认：** 启用

**效果：**

```markdown
这是选中的文本[^1]

[^1]: 这是我的评论
```

**优势：**

- 兼容 Sidebar Highlights 插件
- 更简洁的格式
- 评论紧跟内容

#### 选项 2：捕捉上下文

**默认：** 禁用

**效果：**

```markdown
前面的上下文 ==这是选中的文本== 后面的上下文
```

**适用场景：**

- 选中的文字较短
- 需要保留背景信息
- 想要高亮显示

#### 选项 3：上下文长度

**默认：** 200 字符

**建议：**

- 短句：100-200
- 段落：200-400
- 长文：400-1000

#### 选项 4：上下文模式

**默认：** 按字符数

**选项：**

- **按字符数**：精确控制（推荐）
- **按句子数**：保持句子完整

### 4. 保存设置

点击页面底部的"保存配置"按钮

## 📋 推荐配置

### 配置 A：简洁模式（默认）

**适合：** 大多数用户

```
✅ 使用脚注格式
❌ 捕捉上下文
```

**效果：**

- 只保存选中的文本
- 评论以脚注形式显示
- 最简洁的输出

### 配置 B：上下文模式

**适合：** 需要保留背景信息的用户

```
✅ 使用脚注格式
✅ 捕捉上下文
上下文长度：200
上下文模式：按字符数
```

**效果：**

- 保存选中文本及周围上下文
- 用 `==` 高亮实际选中的部分
- 评论以脚注形式显示

### 配置 C：传统模式

**适合：** 习惯原有格式的用户

```
❌ 使用脚注格式
❌ 捕捉上下文
```

**效果：**

- 使用原有的分隔线和标题格式
- 评论在底部单独显示

## 🎯 使用流程

### 1. 选中文本

在任意网页上选中你想要剪藏的文本

### 2. 右键菜单

右键点击选中的文本 → 选择"剪藏选中内容"

### 3. 添加评论（可选）

在弹出的对话框中：

- 预览选中的内容
- 输入你的评论或笔记
- 点击"剪藏"按钮

### 4. 查看结果

在 Obsidian 中打开生成的笔记，查看效果

## 💡 使用技巧

### 技巧 1：何时使用上下文捕捉？

**推荐使用：**

- ✅ 选中的是一个短句或关键词
- ✅ 需要理解选中文字的背景
- ✅ 想要在 Obsidian 中看到高亮效果

**不推荐使用：**

- ❌ 选中的已经是完整的段落
- ❌ 不需要额外的上下文
- ❌ 追求最简洁的输出

### 技巧 2：调整上下文长度

**太短（< 100）：**

- 可能丢失重要信息
- 上下文不完整

**适中（100-400）：**

- 大多数情况的最佳选择
- 平衡信息量和简洁性

**太长（> 500）：**

- 可能包含过多无关信息
- 笔记变得冗长

### 技巧 3：选择上下文模式

**按字符数：**

- ✅ 精确控制长度
- ✅ 适合大多数情况
- ❌ 可能截断句子

**按句子数：**

- ✅ 保持句子完整
- ✅ 适合文章段落
- ❌ 长度不可控

### 技巧 4：多行评论

在评论框中可以输入多行：

```
重要观点：
1. 第一点
2. 第二点
3. 第三点
```

会自动格式化为：

```markdown
[^1]: 重要观点：

    1. 第一点
    2. 第二点
    3. 第三点
```

## 🎨 在 Obsidian 中的显示

### 脚注格式

在 Obsidian 中：

- 文本后会显示 `[^1]` 上标
- 点击可跳转到脚注
- 脚注在文档底部显示

### 高亮标记

在 Obsidian 中：

- `==文本==` 会显示为黄色高亮
- 可以使用 Sidebar Highlights 插件管理
- 支持导出和搜索

## 🔧 故障排查

### 问题 1：上下文没有捕捉到

**可能原因：**

- 没有启用"捕捉上下文"选项
- 页面结构特殊，无法提取

**解决方案：**

1. 检查设置是否启用
2. 尝试选中更多文本
3. 使用简洁模式

### 问题 2：脚注格式不正确

**可能原因：**

- 没有启用"使用脚注格式"选项
- 评论为空

**解决方案：**

1. 检查设置是否启用
2. 确保输入了评论
3. 重新构建扩展

### 问题 3：高亮不显示

**可能原因：**

- Obsidian 主题不支持
- 语法错误

**解决方案：**

1. 检查 Markdown 语法
2. 尝试其他主题
3. 安装 Sidebar Highlights 插件

## 💬 常见问题

**Q: 脚注格式和传统格式有什么区别？**

A: 脚注格式更简洁，兼容 Sidebar Highlights 插件；传统格式有明显的分隔线和标题。

**Q: 上下文捕捉会影响性能吗？**

A: 不会，上下文提取是轻量级操作，对性能影响可忽略。

**Q: 可以同时使用脚注格式和上下文捕捉吗？**

A: 可以！这是推荐的组合，效果最好。

**Q: 如何在 Obsidian 中管理高亮？**

A: 安装 Sidebar Highlights 插件，可以查看、搜索和导出所有高亮内容。

**Q: 评论可以包含 Markdown 格式吗？**

A: 可以！支持粗体、斜体、列表等基本 Markdown 格式。
