# 🚀 快速开始 - 选中内容剪藏

## 5 分钟上手指南

### 第 1 步：重新加载扩展 (30 秒)

1. 打开 Chrome 扩展管理页面
   ```
   chrome://extensions/
   ```

2. 找到 **"All in Ob"**

3. 点击 **刷新按钮** 🔄

### 第 2 步：确认环境 (1 分钟)

确保以下服务正常运行：

- ✅ **Obsidian** 已打开
- ✅ **Local REST API** 插件已启动
- ✅ 扩展配置正确（可在设置页面测试连接）

### 第 3 步：第一次剪藏 (2 分钟)

#### 操作步骤

1. **打开任意网页**
   - 推荐：Wikipedia、Medium、技术博客

2. **选中一段文字**
   - 用鼠标拖动选中 1-2 个段落
   - 或双击选中一个词

3. **右键点击选中的文字**
   - 会看到菜单项：**"✂️ 剪藏选中内容到 Obsidian"**

4. **在弹出的对话框中**
   - 查看选中内容的预览
   - （可选）添加你的评论，例如：
     ```
     💡 这个观点很有启发性
     ```
   - 点击 **"✂️ 剪藏"** 按钮

5. **等待通知**
   - 浏览器右上角会显示通知
   - 显示文件保存的路径

6. **在 Obsidian 中查看**
   - 打开 Obsidian
   - 找到刚才保存的文件
   - 文件路径类似：
     ```
     Clippings/example.com/2025/2025-10-02/page-title-content-@142345.md
     ```

### 第 4 步：测试精确定位 (1 分钟)

1. 在 Obsidian 中打开刚才的文件

2. 查看 frontmatter 中的 `fragment_url`

3. 按住 Cmd/Ctrl 点击 `fragment_url` 的值
   - 或者复制链接到浏览器

4. 浏览器会：
   - ✅ 打开原网页
   - ✅ 自动滚动到你剪藏的位置
   - ✅ 高亮显示选中的内容（Chrome/Edge）

### 第 5 步：多次剪藏测试 (1 分钟)

在同一个网页上：

1. 选中**第一段文字** → 右键剪藏 → 添加评论 "第一次"
2. 等待 2 秒
3. 选中**第二段文字** → 右键剪藏 → 添加评论 "第二次"
4. 在 Obsidian 中检查：应该有**两个不同的文件**

## 📖 进阶使用

### 使用技巧

#### 1. 快速剪藏工作流

```
浏览 → 选中 → 右键 → 剪藏 → 继续浏览
```

不需要打开 Obsidian，所有内容自动保存。

#### 2. 添加有意义的评论

好的评论示例：
```
💡 这个方法可以应用到我的项目中
🤔 需要验证这个数据的准确性
📌 与《深度工作》一书的观点一致
🔗 相关笔记：[[项目管理方法论]]
```

#### 3. 使用标签分类

在评论中添加标签：
```
这是一个很好的设计模式 #设计模式 #前端开发
```

#### 4. 建立链接网络

在评论中引用其他笔记：
```
这个观点与 [[认知负荷理论]] 相关
可以应用到 [[项目A]] 中
```

### 在 Obsidian 中管理剪藏

#### 查询所有剪藏

使用 Dataview 插件：

```dataview
TABLE 
  page_title as "页面",
  clipped_at as "时间"
FROM ""
WHERE type = "clipper"
SORT clipped_at DESC
LIMIT 20
```

#### 按日期查看

```dataview
LIST
FROM ""
WHERE type = "clipper" 
  AND contains(file.path, "2025-10-02")
```

#### 查找包含特定标签的剪藏

```dataview
TABLE 
  page_title,
  file.link as "文件"
FROM ""
WHERE type = "clipper"
  AND contains(tags, "技术学习")
```

## ❓ 常见问题

### Q1: 对话框没有显示？

**解决方案**：
1. 确认已选中文字
2. 刷新页面后重试
3. 检查浏览器控制台是否有错误

### Q2: 精确定位链接不工作？

**原因**：
- 网站内容已更新
- 浏览器不支持 Text Fragments
- 网站的安全策略限制

**解决方案**：
- 使用 frontmatter 中的 `url` 字段访问原页面
- 手动搜索内容

### Q3: 文件被覆盖了？

**检查**：
1. 确认文件类型是 `clipper`（查看 frontmatter）
2. 检查文件名是否包含时间戳（@HHmmss）
3. 确认不是在同一秒内剪藏的

### Q4: 如何修改文件保存路径？

**步骤**：
1. 点击扩展图标打开设置页面
2. 找到 "Clipper 路径模板" 配置
3. 修改模板，例如：
   ```
   Clipper/{domain}/{yyyy}-{mm}/{slug}.md
   ```
4. 保存配置

### Q5: 可以不添加评论吗？

**可以！**
- 评论是可选的
- 直接点击 "✂️ 剪藏" 按钮即可
- 生成的文件不会包含评论部分
