## 📅 修复日期
2025-10-03

## 🐛 问题描述

### 原始问题

在启用"捕捉上下文"功能后，提取的内容包含了不应该出现的内容：

```
tentLoaded',c);else{var e=document.onreadystatechange||function(){};
document.onreadystatechange=function(b){e(b);'loading'!==document.readyState&&
(document.onreadystatechange=e,c())}}}})();⋮⋮📎 剪藏选中内容 
==funders included Elon Musk, Sam Altman, Reid Hoffman, Peter Thiel
Mission: Ensure AGI benefits all of humanity
Approach: o== 💭 添加你的评论（可选）：取消✂️ 剪藏[^1]
```

### 问题分析

**根本原因：**
1. 使用 `document.body.textContent` 获取全部文本
2. 这会包含所有内容，包括：
   - `<script>` 标签中的 JavaScript 代码
   - `<style>` 标签中的 CSS 代码
   - 隐藏的元素内容
   - 弹窗和对话框的内容

**具体问题：**
- 提取到了页面脚本代码：`tentLoaded',c);else{var e=...`
- 提取到了弹窗内容：`⋮⋮📎 剪藏选中内容 💭 添加你的评论...`
- 上下文不准确，包含了无关内容

## ✅ 解决方案

### 核心改进

1. **从 Selection 对象提取上下文**
   - 不再使用 `document.body.textContent`
   - 直接从用户选中的 Range 对象获取上下文
   - 更准确地定位选中文本的位置

2. **清理文本内容**
   - 移除 `<script>` 标签
   - 移除 `<style>` 标签
   - 移除 `<noscript>` 标签
   - 移除 `<iframe>` 标签

3. **智能容器选择**
   - 查找合适的父容器（段落、div、文章等）
   - 避免提取整个页面的内容
   - 只提取相关的上下文

### 实现细节

#### 1. 新增清理函数

```typescript
function getCleanTextContent(element: Element): string {
  const clone = element.cloneNode(true) as Element;
  
  // Remove script, style, noscript tags
  clone.querySelectorAll('script, style, noscript, iframe').forEach(el => el.remove());
  
  return clone.textContent || '';
}
```

**作用：**
- 克隆元素以避免修改原始 DOM
- 移除所有脚本和样式标签
- 返回干净的文本内容

#### 2. 重写上下文提取函数

**原函数：** `extractContextFromPage(doc, selectedText, config)`

**新函数：** `extractContextFromSelection(selection, config)`

**主要改进：**

```typescript
function extractContextFromSelection(
  selection: Selection,
  config: FragmentClipperConfig

  // 1. 从 selection 获取 range
  const range = selection.getRangeAt(0);
  const selectedText = range.toString().trim();
  
  // 2. 获取公共祖先容器
  let container = range.commonAncestorContainer;
  
  // 3. 如果是文本节点，获取父元素
  if (container.nodeType === Node.TEXT_NODE) {
    container = container.parentElement || container;
  }
  
  // 4. 查找合适的父容器
  let contextContainer: Element | null = container as Element;
  while (contextContainer && contextContainer !== document.body) {
    const tagName = contextContainer.tagName?.toLowerCase();
    if (['p', 'div', 'article', 'section', 'main', 'blockquote', 'li'].includes(tagName)) {
      break;
    }
    contextContainer = contextContainer.parentElement;
  }
  
  // 5. 获取清理后的文本
  const fullText = getCleanTextContent(contextContainer);

  // ...
}
```

**优势：**
- ✅ 只从选中文本所在的容器提取
- ✅ 自动清理脚本和样式
- ✅ 智能选择合适的父容器
- ✅ 避免提取无关内容

## 🔍 技术细节

### 容器选择逻辑

**优先级顺序：**
1. `<p>` - 段落
2. `<div>` - 通用容器
3. `<article>` - 文章
4. `<section>` - 章节
5. `<main>` - 主内容
6. `<blockquote>` - 引用块
7. `<li>` - 列表项

**查找过程：**
```typescript
while (contextContainer && contextContainer !== document.body) {
  const tagName = contextContainer.tagName?.toLowerCase();
  if (['p', 'div', 'article', 'section', 'main', 'blockquote', 'li'].includes(tagName)) {
    break;  // 找到合适的容器
  }
  contextContainer = contextContainer.parentElement;  // 继续向上查找
}
```

### 文本清理过程

**步骤：**
1. 克隆容器元素（避免修改原始 DOM）
2. 查找所有需要移除的标签
3. 移除这些标签
4. 获取清理后的 textContent

**代码：**
```typescript
const clone = element.cloneNode(true) as Element;
clone.querySelectorAll('script, style, noscript, iframe').forEach(el => el.remove());
return clone.textContent || '';
```

### 句子分割改进

**原正则：**
```typescript
const sentences = surroundingText.split(/([.!?。！？]+\s*)/);
```

**新正则：**
```typescript
const sentences = surroundingText.split(/(?<=[.!?。！？])\s+/);
```

**改进：**
- 使用正向后查找（lookbehind）
- 保持标点符号在句子中
- 更准确的句子分割

## 🐛 已知限制

1. **复杂 DOM 结构**
   - 如果选中的文本跨多个容器，可能无法准确提取上下文
   - 解决方案：选择在单个容器内的文本

2. **动态内容**
   - 如果页面内容是动态加载的，可能无法找到容器
   - 解决方案：等待内容加载完成后再选择

3. **特殊字符**
   - 某些特殊字符可能影响文本查找
   - 解决方案：使用简洁模式（不捕捉上下文）
