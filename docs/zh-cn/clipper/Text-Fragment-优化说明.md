# Text Fragment URL 优化说明

## 问题背景

Text Fragment 是一个强大的 Web 标准功能，可以通过 URL 直接定位和高亮网页中的特定文本。但在实际使用中，我们发现并非所有的选中内容都能成功高亮。

### 可以高亮的情况 ✅

```
https://example.com#:~:text=这是一段连续的文本内容
```

**特点**：
- 文本是连续的
- 没有复杂的换行和格式
- 浏览器可以直接匹配

### 不能高亮的情况 ❌

```
https://example.com#:~:text=标题%0A%0A正文内容...
```

**特点**：
- 包含标题和正文
- 有多个段落
- 有复杂的空白字符和换行
- 浏览器无法匹配

## 优化策略

### 核心思路

**使用第一个实质性段落作为 Text Fragment**

原因：
1. 第一个段落通常是连续的文本
2. 避免了跨段落匹配的问题
3. 提高了匹配成功率

### 实现逻辑

```typescript
function generateTextFragmentUrl(baseUrl: string, selectedText: string): string {
  // 1. 按双换行分割段落
  const paragraphs = selectedText
    .split(/\n\s*\n/)
    .map(p => p.trim())
    .filter(p => p.length > 0);
  
  // 2. 找到第一个实质性段落（至少20字符）
  let textToUse = '';
  for (const para of paragraphs) {
    const cleaned = para.replace(/\s+/g, ' ').trim();
    if (cleaned.length >= 20) {
      textToUse = cleaned;
      break;
    }
  }
  
  // 3. 如果没有找到，使用整个文本（清理后）
  if (!textToUse) {
    textToUse = selectedText.replace(/\s+/g, ' ').trim();
  }
  
  // 4. 限制长度（300字符）
  if (textToUse.length > 300) {
    textToUse = textToUse.substring(0, 300);
  }
  
  // 5. URL 编码
  const encodedText = encodeURIComponent(textToUse);
  
  return `${baseUrl}#:~:text=${encodedText}`;
}
```

## 处理流程

### 示例 1：包含标题和正文的选中内容

**原始选中内容**：
```
路径二：让 Agent 自主规划，和它一起成长

另一种就是让 Agent 在自己规定的范围内，尽可能的去自主规划。

这种做法的好处是，当模型能力到了大学，他就能按照大学生的思路重构流程...
```

**处理步骤**：

1. **分割段落**：
   ```
   段落1: "路径二：让 Agent 自主规划，和它一起成长"
   段落2: "另一种就是让 Agent 在自己规定的范围内，尽可能的去自主规划。"
   段落3: "这种做法的好处是..."
   ```

2. **选择第一个实质性段落**：
   - 段落1 长度：19 字符（< 20，跳过）
   - 段落2 长度：30+ 字符（✅ 使用这个）

3. **清理空白字符**：
   ```
   "另一种就是让 Agent 在自己规定的范围内，尽可能的去自主规划。"
   ```

4. **生成 URL**：
   ```
   https://example.com#:~:text=另一种就是让%20Agent%20在自己规定的范围内...
   ```

### 示例 2：连续的长段落

**原始选中内容**：
```
每个领域都有大量的知识，但最宝贵的是什么？是那些领域内的人自己都不知道，都是隐藏知识。这些才是领域的 Memory。一个通用的 Agent，就算能做 Deep Research，它也拿不到这些知识。而一个好的领域 Agent，就是要想办法把这些散落在各处的隐藏知识、暗知识，缓慢地收集起来，变成自己的外脑。
```

**处理步骤**：

1. **分割段落**：
   ```
   段落1: "每个领域都有大量的知识..."（整段）
   ```

2. **选择段落**：
   - 段落1 长度：100+ 字符（✅ 使用这个）

3. **清理空白字符**：
   ```
   "每个领域都有大量的知识，但最宝贵的是什么？是那些领域内的人自己都不知道..."
   ```

4. **限制长度**（如果超过300字符）：
   ```
   "每个领域都有大量的知识，但最宝贵的是什么？是那些领域内的人自己都不知道，都是隐藏知识。这些才是领域的 Memory。一个通用的 Agent，就算能做 Deep Research，它也拿不到这些知识。而一个好的领域 Agent，就是要想办法把这些散落在各处的隐藏知识、暗知识，缓慢地收集起来，变成自己的外脑。"
   ```

5. **生成 URL**：
   ```
   https://example.com#:~:text=每个领域都有大量的知识...
   ```

## 技术细节

### 段落分割规则

使用正则表达式 `/\n\s*\n/` 分割：

```javascript
// 匹配：
// - 换行符
// - 可选的空白字符
// - 再一个换行符

"段落1\n\n段落2"  // ✅ 分割
"段落1\n段落2"    // ❌ 不分割（单换行）
"段落1\n  \n段落2" // ✅ 分割（中间有空格）
```

### 空白字符清理

使用 `.replace(/\s+/g, ' ')` 清理：

```javascript
// 将所有连续的空白字符替换为单个空格

"文本  内容\n换行"  // → "文本 内容 换行"
"多个    空格"      // → "多个 空格"
```

### 最小长度阈值

设置为 **20 个字符**：

**原因**：
- 太短的文本（如标题）可能在页面中重复出现
- 20 字符是一个合理的平衡点
- 确保有足够的上下文进行匹配

### 最大长度限制

设置为 **300 个字符**：

**原因**：
- URL 长度限制（大多数浏览器支持 2000+ 字符）
- 太长的文本可能因为微小差异而匹配失败
- 300 字符足够唯一标识一段文本

## 已知限制

### 1. 动态内容

**问题**：网页内容动态加载或更新

**影响**：Text Fragment 可能无法匹配

**解决方案**：
- 使用 frontmatter 中的 `url` 字段访问原页面
- 手动搜索内容

### 2. 网站限制

**问题**：某些网站的 CSP 策略限制

**影响**：Text Fragment 功能被禁用

**示例**：
- 某些新闻网站
- 企业内部网站

### 3. 浏览器兼容性

**支持情况**：
- ✅ Chrome/Edge 80+
- ✅ Safari 16.1+
- ⚠️ Firefox（需要扩展）

### 4. 文本变化

**问题**：网页内容被编辑

**影响**：原始文本不存在，无法匹配

**解决方案**：
- 剪藏的内容仍然保存在 Obsidian 中
- 可以通过原文链接访问页面

## 最佳实践

### 1. 选择合适的内容

**推荐**：
- ✅ 连续的段落
- ✅ 完整的句子
- ✅ 有一定长度的文本（20+ 字符）

**不推荐**：
- ❌ 只选标题
- ❌ 跨越多个不相关段落
- ❌ 包含大量格式的内容

### 2. 测试精确定位

剪藏后：
1. 在 Obsidian 中打开文件
2. 点击 frontmatter 中的 `fragment_url`
3. 检查是否正确高亮

如果不能高亮：
- 使用 `url` 字段访问原页面
- 手动搜索内容

### 3. 添加上下文

在评论中添加：
```
💡 这段话在文章的第三部分
📍 位置：关于 Agent 规划的讨论
```

## 调试技巧

### 查看生成的 Fragment URL

在 Obsidian 中：
1. 打开剪藏文件
2. 查看 frontmatter 中的 `fragment_url`
3. 复制 URL 到浏览器测试

### 手动构造 Fragment URL

如果自动生成的不工作，可以手动构造：

```
https://example.com#:~:text=要匹配的文本
```

**注意**：
- 文本需要 URL 编码
- 使用连续的文本
- 避免特殊字符

### 使用浏览器开发者工具

1. 打开开发者工具（F12）
2. 在 Console 中测试：
   ```javascript
   // 测试 Text Fragment 是否工作
   window.location.hash = ':~:text=测试文本';
   ```

## 总结

通过优化 Text Fragment URL 的生成策略，我们显著提高了精确定位的成功率。核心思路是：

1. **使用第一个实质性段落**而不是整个选中内容
2. **清理空白字符**确保匹配的一致性
3. **限制长度**避免过长导致的匹配失败
4. **提供回退方案**确保用户总能访问原文

这样的优化让选中内容剪藏功能更加实用和可靠！
