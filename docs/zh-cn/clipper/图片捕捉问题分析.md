# 图片捕捉问题分析：OpenAI vs Gemini

## 问题描述

OpenAI (ChatGPT) 可以成功捕捉聊天中的图片链接，但 Gemini 不行。

## 根本原因分析

### 1. **图片存储方式的差异**

#### ChatGPT 的图片处理方式
- ChatGPT 在 DOM 中使用标准的 `<img>` 标签
- 图片 URL 直接存储在 `src` 属性中
- 示例：
  ```html
  <img src="https://files.oaiusercontent.com/file-xxx/image.png" alt="描述">
  ```

#### Gemini 的图片处理方式
- Gemini 使用 **自定义 Web Components** 来处理图片
- 图片可能存储在以下位置：
  1. **Shadow DOM** 中（无法直接访问）
  2. **Base64 编码**的 data URI
  3. **Blob URLs**（临时 URL，如 `blob:https://gemini.google.com/xxx`）
  4. **Google 内部 API** 返回的临时链接
  5. 可能使用 `<image-query>` 或类似的自定义元素

### 2. **当前代码的图片处理逻辑**

查看 `parse.ts` 中的 `nodeToMarkdown` 函数（第 1294-1299 行）：

```typescript
// Images
if (tagName === 'img') {
  const src = elem.getAttribute('src') || '';
  const alt = elem.getAttribute('alt') || '';
  return `![${alt}](${src})`;
}
```

**问题**：这段代码只能处理标准的 `<img>` 标签，无法处理：
- Shadow DOM 中的图片
- 自定义 Web Components
- Blob URLs 或临时链接

## 可能的解决方案

### 方案 1：增强图片选择器（推荐）

修改 `nodeToMarkdown` 函数，添加对 Gemini 特殊图片元素的支持：

```typescript
// Images - Enhanced for Gemini
if (tagName === 'img') {
  const src = elem.getAttribute('src') || '';
  const alt = elem.getAttribute('alt') || '';
  
  // Skip if src is empty or is a blob URL (temporary)
  if (!src || src.startsWith('blob:')) {
    // Try to find alternative image source
    const dataUrl = elem.getAttribute('data-src') || 
                   elem.getAttribute('data-original-src') ||
                   elem.getAttribute('data-image-url');
    if (dataUrl && !dataUrl.startsWith('blob:')) {
      return `![${alt}](${dataUrl})`;
    }
    // If no valid URL found, skip this image
    return '';
  }
  
  return `![${alt}](${src})`;
}

// Handle Gemini custom image elements
if (tagName === 'image-query' || tagName === 'uploaded-image' || 
    elem.classList.contains('uploaded-image')) {
  // Try to extract image URL from custom element
  const imgElement = elem.querySelector('img');
  if (imgElement) {
    const src = imgElement.getAttribute('src') || 
               imgElement.getAttribute('data-src') || '';
    const alt = imgElement.getAttribute('alt') || 'Image';
    if (src && !src.startsWith('blob:')) {
      return `![${alt}](${src})`;
    }
  }
  
  // Try to get URL from data attributes
  const imageUrl = elem.getAttribute('data-image-url') ||
                  elem.getAttribute('data-src') ||
                  elem.getAttribute('src');
  if (imageUrl && !imageUrl.startsWith('blob:')) {
    return `![Image](${imageUrl})`;
  }
  
  return '';
}
```

### 方案 3：使用 MutationObserver 监听图片加载

对于动态加载的图片，可以使用 MutationObserver：

```typescript
function waitForImages(doc: Document): Promise<void> {
  return new Promise((resolve) => {
    const images = doc.querySelectorAll('img[src^="blob:"]');
    if (images.length === 0) {
      resolve();
      return;
    }
    
    let loadedCount = 0;
    const observer = new MutationObserver((mutations) => {
      mutations.forEach((mutation) => {
        if (mutation.type === 'attributes' && mutation.attributeName === 'src') {
          const img = mutation.target as HTMLImageElement;
          if (!img.src.startsWith('blob:')) {
            loadedCount++;
            if (loadedCount >= images.length) {
              observer.disconnect();
              resolve();
            }
          }
        }
      });
    });
    
    images.forEach(img => {
      observer.observe(img, { attributes: true });
    });
    
    // Timeout after 5 seconds
    setTimeout(() => {
      observer.disconnect();
      resolve();
    }, 5000);
  });
}
```
