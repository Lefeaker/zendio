# 📝 选中内容剪藏 - 最终格式说明

## 文件格式

### YAML Frontmatter

```yaml
---
type: clipper
title: "页面标题"
url: "https://example.com/article#:~:text=选中的内容..."
clipped_at: "2025-10-02T04:06:38"
tags: [clipping, fragment]
---
```

**字段说明**：

- **type**: 固定为 `clipper`，标识这是选中内容剪藏
- **title**: 页面的原始标题（不包含时间戳）
- **url**: Text Fragment URL（精确定位链接，点击可直接跳转到选中内容）
- **clipped_at**: 剪藏时间，格式为 `YYYY-MM-DDTHH:mm:ss`（24小时制）
- **tags**: 标签数组，默认包含 `clipping` 和 `fragment`

### 正文内容

```markdown
剪藏的内容直接展示，已转换为 Markdown 格式...

---

## 💭 我的评论

你添加的评论内容...
```

**说明**：
- 剪藏内容直接展示，无额外标题
- 如果添加了评论，会在内容下方显示
- 评论和内容之间用分隔线（`---`）隔开

## 文件命名

### 格式

```
{页面标题} {YYYY-MM-DDTHH:mm:ss}.md
```

### 示例

```
React Documentation 2025-10-02T14:23:45.md
深度学习入门 2025-10-02T16:30:12.md
产品设计原则 2025-10-02T09:15:33.md
```

### 特点

- ✅ **简洁明了**：只包含页面标题和时间
- ✅ **唯一性**：精确到秒，避免覆盖
- ✅ **易于排序**：按时间自然排序
- ✅ **易于识别**：一眼就能看出内容和时间

## 格式优势

### 1. 简洁性 ✅

**YAML 只包含必要信息**：
- 类型标识
- 页面标题
- 精确定位链接
- 剪藏时间
- 标签

**去除了冗余字段**：
- ❌ 不再有 `page_title`（与 `title` 重复）
- ❌ 不再有 `fragment_url`（已合并到 `url`）
- ❌ 不再有 ISO 格式的时间（改用更易读的格式）

### 2. 易读性 ✅

**时间格式更友好**：
```
旧格式：2025-10-02T14:23:45.000Z
新格式：2025-10-02T14:23:45
```

**文件名更简洁**：
```
旧格式：React Documentation - useState is a Hook... @142345.md
新格式：React Documentation 2025-10-02T14:23:45.md
```

### 3. 实用性 ✅

**URL 直接可用**：
- 点击 `url` 字段就能跳转到精确位置
- 不需要区分 `url` 和 `fragment_url`

**文件名易于管理**：
- 按时间自然排序
- 易于搜索和过滤
- 不会太长

## 在 Obsidian 中使用

### 查看精确定位链接

1. 打开剪藏文件
2. 在 frontmatter 中找到 `url` 字段
3. Cmd/Ctrl + 点击链接
4. 浏览器会打开原网页并高亮显示内容

### 使用 Dataview 查询

**查询所有剪藏**：
```dataview
TABLE 
  title as "标题",
  clipped_at as "时间"
FROM ""
WHERE type = "clipper"
SORT clipped_at DESC
```

**查询今天的剪藏**：
```dataview
LIST
FROM ""
WHERE type = "clipper" 
  AND contains(clipped_at, "2025-10-02")
```

**按标题搜索**：
```dataview
TABLE 
  clipped_at as "时间",
  file.link as "文件"
FROM ""
WHERE type = "clipper"
  AND contains(title, "React")
```
