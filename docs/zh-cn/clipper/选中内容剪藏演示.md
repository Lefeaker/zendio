# 📎 选中内容剪藏功能演示

## 功能亮点

这个新功能让你可以像使用 **Notion Web Clipper** 或 **Hypothesis** 一样，快速收集网页中的精彩片段，并添加你的个人思考。

### 🎯 核心优势

1. **精确定位**：生成的链接可以直接跳转到网页的特定位置
2. **个人评论**：在剪藏时添加你的想法和笔记
3. **碎片化收集**：不需要保存整个页面，只收集有价值的部分
4. **优雅界面**：美观的对话框设计，提升使用体验

## 使用场景示例

### 场景 1：阅读技术文章

你在阅读一篇关于 React 的文章，发现了一个很有用的代码片段：

```
1. 选中代码片段
2. 右键 → "✂️ 剪藏选中内容到 Obsidian"
3. 添加评论："这个 Hook 的实现很巧妙，可以用在我的项目中"
4. 点击剪藏
```

**结果**：在 Obsidian 中得到一个包含代码、评论和精确链接的笔记。

### 场景 2：收集灵感

浏览设计网站时，看到一段关于设计原则的描述：

```
1. 选中文字
2. 右键剪藏
3. 添加评论："可以应用到我们的产品设计中 #设计灵感"
4. 保存
```

**结果**：快速建立你的灵感库，每条都有来源和你的思考。

### 场景 3：学术研究

阅读论文时，发现重要的研究结论：

```
1. 选中关键段落
2. 右键剪藏
3. 添加评论："这个数据支持我的假设，需要在论文中引用"
4. 保存
```

**结果**：建立你的研究笔记库，每条都可以追溯到原文。

### 生成的 Markdown 文件

```markdown
---
type: clipper
title: '文章标题'
url: 'https://example.com/article'
fragment_url: 'https://example.com/article#:~:text=选中的内容'
clipped_at: '2025-10-02T12:00:00'
tags: [clipping]
---

# 文章标题

> [!info] 来源
> 📎 [精确定位](fragment_url) ← 点击直接跳转到原文位置
> 🔗 [原文链接](url)
> 📅 2025-10-02T12:00:00

## 💭 我的评论

这里是你添加的评论和思考...

## 📝 剪藏内容

这里是选中的内容，已转换为 Markdown 格式...

---

_通过 All in Ob 剪藏_
```

## 高级技巧

### 1. 使用标签系统

在评论中添加标签，方便后续检索：

```
💡 这个观点很有启发性 #创新 #产品设计
```

### 2. 建立链接网络

在评论中引用其他笔记：

```
这个观点与 [[设计原则]] 笔记中的内容相呼应
```

### 3. 使用 Dataview 查询

在 Obsidian 中使用 Dataview 插件查询所有剪藏：

```dataview
TABLE
  title as "标题",
  clipped_at as "剪藏时间"
FROM ""
WHERE type = "clipper"
SORT clipped_at DESC
```

### 4. 创建每日剪藏汇总

使用 Templater 插件自动汇总当天的剪藏：

```javascript
<%*
const today = moment().format('YYYY-MM-DD');
const clippings = dv.pages()
  .where(p => p.type === 'clipper' && p.clipped_at.startsWith(today));
%>
```
