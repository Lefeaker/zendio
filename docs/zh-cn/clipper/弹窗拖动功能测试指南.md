# 弹窗拖动功能测试指南

## 🎯 测试目标

验证剪藏弹窗的拖动功能是否正常工作，包括：
- 基本拖动功能
- 视觉反馈
- 触摸支持
- 边界情况

## 📋 测试前准备

### 1. 构建扩展

```bash
cd /Users/mac/Documents/Dev/AI2OB_Plg/AiiinOB/your-extension
npm run build
```

### 2. 重新加载扩展

1. 打开 Chrome，访问 `chrome://extensions/`
2. 找到 "All in Ob" 扩展
3. 点击"重新加载"按钮（圆形箭头图标）

### 3. 准备测试页面

打开任意包含文本的网页，例如：
- 新闻网站
- 博客文章
- 维基百科

## ✅ 测试用例

### 测试 1: 基本拖动功能

**步骤：**
1. 在网页上选中一段文本
2. 右键点击，选择"剪藏选中内容"
3. 观察弹窗出现在屏幕中央
4. 将鼠标移动到标题栏（"剪藏到 Obsidian"）
5. 观察光标变为 `move` 样式（四向箭头）
6. 按住鼠标左键并拖动
7. 观察弹窗跟随鼠标移动
8. 释放鼠标
9. 观察弹窗停留在新位置

**预期结果：**
- ✅ 弹窗可以流畅拖动
- ✅ 光标显示为 move 样式
- ✅ 弹窗跟随鼠标移动
- ✅ 释放后停留在新位置

### 测试 2: 视觉反馈

**步骤：**
1. 打开剪藏弹窗
2. 将鼠标悬停在标题栏上（不点击）
3. 观察标题栏背景变化
4. 点击并开始拖动
5. 观察拖动时的视觉变化
6. 释放鼠标
7. 观察视觉恢复正常

**预期结果：**
- ✅ 悬停时标题栏背景变亮
- ✅ 拖动时阴影增强（更深、更大）
- ✅ 拖动时标题栏背景高亮
- ✅ 释放后视觉恢复正常

### 测试 3: 拖动图标

**步骤：**
1. 打开剪藏弹窗
2. 观察标题文字前面的图标
3. 确认显示 `⋮⋮` 图标

**预期结果：**
- ✅ 显示拖动图标 `⋮⋮`
- ✅ 图标颜色为灰色，半透明
- ✅ 图标提示用户可以拖动

### 测试 4: 拖动范围

**步骤：**
1. 打开剪藏弹窗
2. 尝试将弹窗拖动到屏幕各个位置：
   - 左上角
   - 右上角
   - 左下角
   - 右下角
   - 屏幕中央
   - 屏幕边缘

**预期结果：**
- ✅ 可以拖动到任意位置
- ⚠️ 注意：当前版本没有边界限制，可能拖出屏幕

### 测试 5: 交互元素不触发拖动

**步骤：**
1. 打开剪藏弹窗
2. 尝试点击并拖动以下元素：
   - 输入框（textarea）
   - "取消"按钮
   - "剪藏"按钮
   - 预览区域的文本

**预期结果：**
- ✅ 点击输入框不会触发拖动
- ✅ 点击按钮不会触发拖动
- ✅ 只有标题栏可以拖动

### 测试 6: 拖动后功能正常

**步骤：**
1. 打开剪藏弹窗
2. 拖动弹窗到新位置
3. 在输入框中输入文本
4. 点击"剪藏"按钮

**预期结果：**
- ✅ 拖动后输入框仍然可用
- ✅ 拖动后按钮仍然可点击
- ✅ 剪藏功能正常工作

### 测试 7: 页面交互不受影响

**步骤：**
1. 打开剪藏弹窗
2. 不关闭弹窗，尝试：
   - 点击页面其他区域
   - 滚动页面
   - 选择页面文本
   - 点击页面链接

**预期结果：**
- ✅ 可以点击页面内容
- ✅ 可以滚动页面
- ✅ 可以选择文本
- ✅ 弹窗不会阻止页面交互

### 测试 8: ESC 键关闭

**步骤：**
1. 打开剪藏弹窗
2. 拖动到任意位置
3. 按 ESC 键

**预期结果：**
- ✅ 弹窗正常关闭
- ✅ 拖动不影响 ESC 键功能

### 测试 9: 多次打开位置

**步骤：**
1. 打开剪藏弹窗
2. 拖动到屏幕右侧
3. 关闭弹窗
4. 再次打开剪藏弹窗

**预期结果：**
- ✅ 新弹窗出现在屏幕中央（默认位置）
- ℹ️ 注意：当前版本不记忆上次位置

### 测试 10: 触摸拖动（移动设备/触摸屏）

**步骤：**
1. 在触摸设备上打开剪藏弹窗
2. 用手指触摸标题栏
3. 拖动弹窗
4. 释放手指

**预期结果：**
- ✅ 触摸拖动流畅
- ✅ 视觉反馈正常
- ✅ 释放后停留在新位置

## 🐛 已知问题和限制

### 当前版本的限制

1. **无边界限制**
   - 弹窗可以被拖出屏幕
   - 如果拖出屏幕，需要刷新页面重置

2. **不记忆位置**
   - 每次打开都在屏幕中央
   - 不记住上次拖动的位置

3. **多屏幕支持**
   - 可以拖到副屏幕
   - 但可能超出可见范围

### 未来可能的改进

1. **边界检测**
   ```typescript
   // 限制在视口内
   const maxX = window.innerWidth - dialogWidth;
   const maxY = window.innerHeight - dialogHeight;
   ```

2. **位置记忆**
   ```typescript
   // 保存到 localStorage
   localStorage.setItem('dialogPosition', JSON.stringify({ x, y }));
   ```

3. **智能定位**
   - 避开选中的文本
   - 自动选择最佳位置

4. **双击重置**
   - 双击标题栏回到中央

## 🔧 问题排查

### 如果拖动不工作

1. **检查构建**
   ```bash
   npm run build
   ```

2. **检查控制台错误**
   - 打开 DevTools (F12)
   - 查看 Console 标签
   - 查找红色错误信息

3. **重新加载扩展**
   - 访问 `chrome://extensions/`
   - 点击"重新加载"

4. **清除缓存**
   - 关闭所有标签页
   - 重启浏览器

### 如果弹窗被拖出屏幕

**临时解决方案：**
1. 刷新页面
2. 重新打开弹窗（会回到中央）

**永久解决方案：**
- 等待边界检测功能实现
