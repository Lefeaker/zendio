# 无痕模式问题解决方案

## 🔍 问题原因

扩展在无痕窗口中显示为灰色（禁用状态）的**根本原因**是：

### Chrome 扩展图标显示规则
在 Chrome 中，扩展图标会显示为灰色（禁用）如果：
1. 扩展的 `action` 没有定义 `default_popup`（弹出页面）
2. **并且** 没有 `chrome.action.onClicked` 监听器

你的扩展之前只有右键菜单功能，没有图标点击处理，所以 Chrome 认为这个扩展在当前页面"不可用"，因此显示为灰色。

**这与无痕模式无关！** 在普通窗口中图标也应该是灰色的（除非你之前没注意到）。

## ✅ 解决方案

我已经做了以下修改：

### 1. 添加了 `incognito` 配置
在 `manifest.json` 中添加：
```json
"incognito": "spanning"
```

这样普通窗口和无痕窗口会共享配置和存储。

### 2. 添加了图标点击处理器
在 `src/background/index.ts` 中添加：
```typescript
// Handle extension icon click
chrome.action.onClicked.addListener(async (tab) => {
  if (!tab?.id) return;

  try {
    // Inject content script to clip the full page
    await chrome.scripting.executeScript({
      target: { tabId: tab.id },
      files: ['content/index.js'],
      world: 'ISOLATED'
    });
  } catch (error) {
    console.error('Failed to inject content script:', error);
    chrome.notifications.create({
      type: 'basic',
      iconUrl: 'assets/icons/icon128.png',
      title: '剪藏失败',
      message: '无法注入内容脚本: ' + (error as Error).message
    });
  }
});
```

## 🚀 现在的功能

扩展现在有**三种使用方式**：

### 方式 1：点击扩展图标 🆕
- 点击工具栏中的扩展图标
- 自动剪藏当前整个页面

### 方式 2：右键菜单 - 整页剪藏
- 在页面任意位置右键
- 选择 "剪藏整个页面到 Obsidian"

### 方式 3：右键菜单 - 选中内容剪藏
- 选中页面中的文字
- 右键选择 "✂️ 剪藏选中内容到 Obsidian"
- 可以添加评论

## 📋 使用步骤

1. 打开 `chrome://extensions/`
2. 找到 "All in Obsidian"

### 2. 测试普通窗口
1. 打开任意网页
2. 查看扩展图标 - 应该**不再是灰色**
3. 点击图标测试剪藏功能

### 3. 测试无痕窗口
1. 打开无痕窗口（Ctrl/Cmd + Shift + N）
2. 打开任意网页
3. 查看扩展图标 - 应该**不再是灰色**
4. 点击图标测试剪藏功能
5. 配置应该与普通窗口一致（因为使用了 `spanning` 模式）

## 🎯 关键点

### 为什么之前是灰色的？
- Chrome 认为没有点击处理的扩展在当前页面"不可用"
- 所以显示为灰色（禁用状态）
- 这是 Chrome 的 UI 设计，不是 bug

### 为什么现在不是灰色了？
- 添加了 `chrome.action.onClicked` 监听器
- Chrome 知道点击图标会执行操作
- 所以显示为正常颜色（启用状态）

### `incognito: "spanning"` 的作用
- 让普通窗口和无痕窗口共享：
  - 存储数据（配置、设置等）
  - Service Worker 实例
  - 扩展状态
- 你只需要配置一次 Obsidian 连接信息

## 🔧 如果还有问题

### 问题 A：图标仍然是灰色
**可能原因**：
1. 没有重新加载扩展
2. 在受限页面上（chrome://, edge://, about: 等）

**解决方法**：
1. 确保重新加载了扩展
2. 在普通网页上测试（如 https://www.baidu.com）

### 问题 B：点击图标没反应
**可能原因**：
1. Service Worker 未启动
2. 页面 URL 不支持脚本注入

**解决方法**：
1. 打开 `chrome://extensions/`
2. 点击 "Service Worker" 查看控制台
3. 查看是否有错误信息

### 问题 C：无痕窗口中配置丢失
**可能原因**：
- 这不应该发生，因为使用了 `spanning` 模式

**解决方法**：
1. 确认 manifest.json 中有 `"incognito": "spanning"`
2. 重新加载扩展
3. 检查 Service Worker 控制台是否有错误

## 📝 技术细节

### Manifest V3 的 Action API
```json
{
  "action": {
    "default_title": "Clip to Obsidian"
  }
}
```

- 如果只有 `default_title`，没有 `default_popup` 或点击监听器
- Chrome 会认为这个 action 不可用
- 图标显示为灰色

### 解决方案
添加以下任一项：
1. `default_popup`: 点击时显示弹出页面
2. `chrome.action.onClicked`: 点击时执行代码

我们选择了方案 2，让点击图标直接剪藏页面。
