# 无痕模式调试指南

## 调试步骤

### 1. 检查 Service Worker 状态

1. **在普通窗口中**：
   - 打开 `chrome://extensions/`
   - 找到 "All in Ob"
   - 点击 "Service Worker" 链接（如果显示为 "inactive"，点击它激活）
   - 查看控制台是否有错误

2. **在无痕窗口中**：
   - 打开 `chrome://extensions/`
   - 检查扩展是否真的启用了（开关是否打开）
   - 点击 "详细信息"
   - 确认 "在无痕模式下启用" 是打开的

### 2. 检查扩展图标状态

在无痕窗口中：
1. 打开任意网页（例如 https://www.baidu.com）
2. 右键点击页面
3. 查看右键菜单中是否有：
   - "剪藏整个页面到 Obsidian"
   - "✂️ 剪藏选中内容到 Obsidian"（选中文字后）

### 3. 查看控制台错误

**在无痕窗口中打开开发者工具**：
1. 按 F12 或 Ctrl+Shift+I (Mac: Cmd+Option+I)
2. 切换到 "Console" 标签
3. 尝试使用扩展功能
4. 查看是否有错误信息

### 4. 检查权限

在 `chrome://extensions/` 中：
1. 点击扩展的 "详细信息"
2. 向下滚动查看 "权限" 部分
3. 确认以下权限已授予：
   - 读取和更改您访问的网站上的所有数据
   - 显示通知
   - 存储数据

### 5. 可能的问题和解决方案

#### 问题 A: Service Worker 未在无痕模式下启动
**症状**：右键菜单没有扩展选项
**解决**：
1. 在无痕窗口打开 `chrome://extensions/`
2. 找到扩展，点击 "Service Worker"
3. 查看控制台是否有错误

#### 问题 B: 动态脚本注入失败
**症状**：右键菜单有选项，但点击后没有反应
**解决**：
1. 检查页面 URL 是否是受限页面（chrome://, edge://, about: 等）
2. 尝试在普通网页上测试（如 https://www.baidu.com）

#### 问题 C: 存储权限问题
**症状**：扩展运行但无法读取配置
**解决**：
1. 在无痕窗口打开扩展的设置页面
2. 检查配置是否存在（如果使用 spanning 模式，应该能看到普通窗口的配置）

### 6. 测试命令

在无痕窗口的控制台中运行以下命令来测试：

```javascript
// 测试扩展是否可访问
chrome.runtime.id

// 测试存储是否可访问
chrome.storage.local.get(null, (data) => console.log('Storage:', data))

// 测试消息传递
chrome.runtime.sendMessage({type: 'TEST'}, (response) => console.log('Response:', response))
```

### 7. 收集信息

如果问题仍然存在，请收集以下信息：

1. **Chrome 版本**：在地址栏输入 `chrome://version/`
2. **扩展版本**：0.1.0
3. **错误信息**：从 Service Worker 控制台和页面控制台复制所有错误
4. **测试页面 URL**：你在哪个网页上测试的
5. **右键菜单**：是否能看到扩展的右键菜单选项

## 常见原因

### 1. Chrome 策略限制
某些企业或学校的 Chrome 可能有策略限制扩展在无痕模式下运行。

检查方法：
- 打开 `chrome://policy/`
- 搜索 "incognito" 或 "extension"
- 查看是否有限制性策略

### 2. 扩展冲突
其他扩展可能干扰了你的扩展。

测试方法：
- 禁用所有其他扩展
- 只保留 "All in Ob"
- 重新测试

### 3. 缓存问题
扩展的旧版本可能被缓存。

解决方法：
1. 完全卸载扩展
2. 重启 Chrome
3. 重新加载扩展
