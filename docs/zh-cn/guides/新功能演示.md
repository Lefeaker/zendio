### 2. 域名映射功能 🌐

这是本次更新的核心功能！

#### 问题场景
以前保存微信公众号文章时，路径是这样的：
```
Clippings/mp.weixin.qq.com/2024/article-title.md
```

这样的问题：
- 域名太长，不直观
- 文件夹名称不友好
- 难以快速识别来源

#### 解决方案
现在可以设置域名映射：
```
mp.weixin.qq.com → 公众号
```

保存后的路径变成：
```
Clippings/公众号/2024/article-title.md
```

#### 预设映射
系统已经预设了常用网站的映射：

| 原域名 | 映射后 | 说明 |
|--------|--------|------|
| mp.weixin.qq.com | 公众号 | 微信公众号文章 |
| wx.zsxq.com | 知识星球 | 知识星球内容 |
| www.zhihu.com | 知乎 | 知乎问答 |
| juejin.cn | 掘金 | 掘金技术文章 |
| www.youtube.com | YouTube | YouTube 视频 |
| twitter.com | Twitter | Twitter 推文 |
| medium.com | Medium | Medium 文章 |

#### 自定义映射
你可以添加任何网站的映射：

**示例 1：个人博客**
```
blog.example.com → 我的博客
```

**示例 2：公司文档**
```
docs.company.com → 公司文档
wiki.company.com → 公司Wiki
```

**示例 3：学习资源**
```
www.coursera.org → Coursera课程
www.udemy.com → Udemy课程
```

### 3. 改进的模板配置 📝

#### 更清晰的变量说明
每个模板输入框下方都有可用变量的提示：

**文章模板：**
```
可用变量：{domain} {yyyy} {mm} {dd} {slug} {title}
```

**AI 对话模板：**
```
可用变量：{platform} {yyyy} {mm} {dd} {title}
```

#### 实际应用示例

**场景 1：按来源分类**
```
模板：Clippings/{domain}/{yyyy}/{slug}.md
映射：mp.weixin.qq.com → 公众号

结果：Clippings/公众号/2024/my-article.md
```

**场景 2：按时间归档**
```
模板：Archive/{yyyy}/{mm}/{domain}/{slug}.md
映射：www.zhihu.com → 知乎

结果：Archive/2024/01/知乎/question-answer.md
```

**场景 3：扁平化结构**
```
模板：Notes/{domain}_{yyyy}{mm}{dd}_{slug}.md
映射：juejin.cn → 掘金

结果：Notes/掘金_20240115_article-title.md
```

## 🚀 快速开始

3. 尝试剪藏一篇文章验证效果

## 💡 使用技巧

### 技巧 1：为不同类型内容设置不同结构

**技术文章：**
```
模板：Tech/{domain}/{yyyy}/{slug}.md
映射：
  juejin.cn → 掘金
  www.infoq.cn → InfoQ
```

**新闻资讯：**
```
模板：News/{yyyy}-{mm}/{domain}/{slug}.md
映射：
  www.36kr.com → 36氪
  www.huxiu.com → 虎嗅
```

### 技巧 2：使用中文文件夹名

域名映射支持中文，让你的笔记库更加本土化：
```
mp.weixin.qq.com → 公众号
www.zhihu.com → 知乎
wx.zsxq.com → 知识星球
```

### 技巧 3：保持映射简洁

映射名称建议：
- ✅ 简短易记：`公众号` 而不是 `微信公众号文章`
- ✅ 统一风格：都用中文或都用英文
- ✅ 避免特殊字符：不要使用 `/` `\` `:` 等

### 技巧 4：定期整理映射

随着使用，你可能会积累很多映射：
- 删除不再使用的网站映射
- 合并相似的映射
- 保持列表简洁
