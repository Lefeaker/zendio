# 📝 文件命名说明

## 概述

为了避免在同一页面多次剪藏时文件被覆盖，不同类型的剪藏使用不同的文件命名策略。

## 文件命名规则

### 1. 完整文章剪藏 (article)

**默认模板**：`Clippings/{domain}/{yyyy}/{slug}.md`

**命名方式**：
- 使用文章标题的 slug 化版本
- 同一篇文章多次剪藏会覆盖（这是预期行为）

**示例**：
```
原标题：Understanding React Hooks
文件名：understanding-react-hooks.md
完整路径：Clippings/medium.com/2025/understanding-react-hooks.md
```

### 2. 选中内容剪藏 (clipper)

**默认模板**：`Clippings/{domain}/{yyyy}/{yyyy}-{mm}-{dd}/{slug}.md`

**命名方式**：
- 使用 **页面标题 + 选中内容预览 + 时间戳** 生成唯一标题
- 按日期分组，每天的剪藏放在独立的文件夹中
- 每个剪藏都有唯一的文件名，不会覆盖

**标题格式**：
```
{页面标题} - {选中内容前30字}... @{时间戳}
```

**时间戳格式**：`HHmmss`（时分秒）

**示例 1**：
```
页面标题：React Documentation
选中内容：useState is a Hook that lets you add React state...
剪藏时间：14:23:45

生成标题：React Documentation - useState is a Hook that let... @142345
文件名：react-documentation-usestate-is-a-hook-that-let-@142345.md
完整路径：Clippings/react.dev/2025/2025-10-02/react-documentation-usestate-is-a-hook-that-let-@142345.md
```

**示例 2**（同一页面多次剪藏）：
```
第一次剪藏（14:23:45）：
  Clippings/react.dev/2025/2025-10-02/react-documentation-usestate-is-a-hook-@142345.md

第二次剪藏（14:25:30）：
  Clippings/react.dev/2025/2025-10-02/react-documentation-useeffect-lets-you-@142530.md

第三次剪藏（14:27:15）：
  Clippings/react.dev/2025/2025-10-02/react-documentation-custom-hooks-let-@142715.md
```

### 3. AI 对话剪藏 (ai_chat)

**默认模板**：`AI/{platform}/{yyyy}/{yyyy}-{mm}-{dd}_{title}.md`

**命名方式**：
- 使用日期 + 对话标题
- 同一个对话多次剪藏会覆盖（这是预期行为）

**示例**：
```
平台：ChatGPT
标题：How to learn TypeScript
文件名：2025-10-02_how-to-learn-typescript.md
完整路径：AI/ChatGPT/2025/2025-10-02_how-to-learn-typescript.md
```

## 自定义模板

你可以在设置页面自定义 clipper 类型的路径模板。

### 可用变量

- `{domain}` - 网站域名（可通过域名映射转换）
- `{yyyy}` - 年份（4位）
- `{mm}` - 月份（2位，补零）
- `{dd}` - 日期（2位，补零）
- `{slug}` - 标题的 slug 化版本（小写，空格转连字符）

### 推荐模板

#### 模板 1：按日期分组（默认）
```
Clippings/{domain}/{yyyy}/{yyyy}-{mm}-{dd}/{slug}.md
```

**优点**：
- ✅ 按日期清晰分组
- ✅ 易于查找特定日期的剪藏
- ✅ 文件夹结构清晰

**示例**：
```
Clippings/medium.com/2025/2025-10-02/article-title-preview-@142345.md
```

#### 模板 2：扁平化结构
```
Clippings/{domain}/{yyyy}/{slug}.md
```

**优点**：
- ✅ 结构简单
- ✅ 所有剪藏在同一层级

**缺点**：
- ⚠️ 同一天的多个剪藏混在一起

**示例**：
```
Clippings/medium.com/2025/article-title-preview-@142345.md
```

#### 模板 3：按月分组
```
Clippings/{domain}/{yyyy}/{yyyy}-{mm}/{slug}.md
```

**优点**：
- ✅ 按月分组，不会太细
- ✅ 适合剪藏量适中的情况

**示例**：
```
Clippings/medium.com/2025/2025-10/article-title-preview-@142345.md
```

#### 模板 4：独立的 Clipper 目录
```
Clipper/{domain}/{yyyy}-{mm}-{dd}/{slug}.md
```

**优点**：
- ✅ 与完整文章剪藏完全分离
- ✅ 更容易区分剪藏类型

**示例**：
```
Clipper/medium.com/2025-10-02/article-title-preview-@142345.md
```

## 文件内容结构

### Frontmatter 字段

```yaml
---
type: clipper                    # 类型标识
title: "页面标题 - 内容预览... @142345"  # 唯一标题
page_title: "页面标题"            # 原始页面标题
url: "https://..."               # 原始URL
fragment_url: "https://...#:~:text=..."  # 精确定位URL
clipped_at: "2025-10-02T14:23:45.000Z"   # 剪藏时间
tags: [clipping, fragment]       # 标签
---
```

### 关键字段说明

- **title**：用于生成文件名的唯一标题
- **page_title**：原始页面标题，用于显示
- **fragment_url**：可以直接定位到选中内容的URL

## 最佳实践

### 1. 选择合适的模板

根据你的使用习惯选择：

- **频繁剪藏**：使用按日期分组的模板
- **偶尔剪藏**：使用按月分组或扁平化模板
- **严格分类**：使用独立的 Clipper 目录

### 2. 使用域名映射

配合域名映射功能，让路径更友好：

```
配置映射：
  medium.com → Medium
  dev.to → DevTo

结果：
  Clippings/Medium/2025/2025-10-02/article-@142345.md
  Clippings/DevTo/2025/2025-10-02/tutorial-@143012.md
```

### 3. 利用标签

在 Obsidian 中使用标签查询：

```dataview
LIST
FROM ""
WHERE type = "clipper"
AND contains(tags, "fragment")
SORT clipped_at DESC
LIMIT 10
```

### 4. 定期整理

建议定期整理剪藏：

- 删除不需要的片段
- 将有价值的内容整合到主题笔记
- 添加更多的标签和链接

## 文件名长度限制

为了兼容不同的文件系统，文件名会被限制在 180 个字符以内。

如果生成的文件名过长，会自动截断：

```
原标题：React Documentation - useState is a Hook that lets you add React state to your components @142345
截断后：react-documentation-usestate-is-a-hook-that-lets-you-add-react-state-to-your-components-142345（如果超过180字符会被截断）
```

## 特殊字符处理

文件名中的特殊字符会被替换为下划线：

```
原字符：\ / : * ? " < > |
替换为：_
```

示例：
```
原标题：What's new in React 18?
文件名：what_s-new-in-react-18_.md
```

## 时间戳说明

### 为什么使用时间戳？

1. **唯一性**：确保同一秒内的剪藏也不会冲突
2. **可读性**：HHmmss 格式易于理解
3. **排序性**：按时间自然排序

### 时间戳格式

```
格式：HHmmss
H：小时（00-23）
m：分钟（00-59）
s：秒（00-59）

示例：
14:23:45 → 142345
09:05:03 → 090503
23:59:59 → 235959
```

## 故障排除

### 问题：文件名太长

**解决方案**：
- 选中的内容预览会自动限制在 30 个字符
- 整个文件名会限制在 180 个字符
- 如果还是太长，考虑使用更短的域名映射

### 问题：文件仍然被覆盖

**检查**：
1. 确认是 clipper 类型（查看 frontmatter）
2. 检查时间戳是否正确生成
3. 查看浏览器控制台的日志

**可能原因**：
- 在同一秒内多次剪藏（极少见）
- 模板配置错误

### 问题：找不到剪藏的文件

**解决方案**：
1. 在 Obsidian 中搜索 `type:clipper`
2. 检查日期文件夹
3. 使用 Dataview 查询所有剪藏

## 总结

| 类型 | 文件名策略 | 是否唯一 | 适用场景 |
|------|-----------|---------|---------|
| article | 基于文章标题 | ❌ 会覆盖 | 完整文章，一次性剪藏 |
| clipper | 标题+预览+时间戳 | ✅ 唯一 | 碎片化内容，多次剪藏 |
| ai_chat | 日期+对话标题 | ❌ 会覆盖 | AI 对话，更新式剪藏 |
