# 设置页面使用指南

## 🎨 新版设置页面特性

新版设置页面采用了现代化的深色主题设计，提供了更直观、更强大的配置选项。

## 📋 主要功能

### 1. Obsidian Local REST API 配置

配置与 Obsidian 的连接：

- **HTTPS URL**: 安全连接地址（默认端口 27124）
- **HTTP URL**: 备用连接地址（默认端口 27123）
- **Vault 名称**: 你的 Obsidian 仓库名称
- **本地目录**: Chromium 浏览器中可选的 Vault 文件夹授权。配置后优先直接写入本地目录；Chrome 重启后如果权限变为待确认，下一次剪藏或导出会在当前页面请求重新授权，允许后当次操作继续写入本地目录，选择不再询问则后续回退 REST API。
- **API Key**: 在 Obsidian Local REST API 插件中获取

推荐优先使用本地目录通道。REST API 功能由 Obsidian 插件 [Local REST API with MCP](https://github.com/coddingtonbear/obsidian-local-rest-api) 支持。

插件会智能选择可用的连接方式，优先使用 HTTPS。Vault List 右上角的“测试连接”会逐个检查启用仓库的本地目录、HTTPS 与 HTTP 通道，并用 ✅ / ❌ 分别标出结果；如果 HTTPS 失败疑似本地证书未信任，结果中会提供对应的证书下载与信任链接。

### 1.1 本地 Vault 目录（可选）

在 Chromium 浏览器中，你可以为每个 Vault 选择一个本地目录，让扩展通过浏览器 File System Access 写入文件：

- 必须由用户在设置页主动点击并选择目录；扩展不会自动扫描本地文件夹。
- 写入范围限制在浏览器授权的目录句柄内，保存路径仍会按你的模板生成并做安全校验。
- 如果目录权限缺失、被拒绝、浏览器不支持、预检失败或写入失败，且 REST API 已配置，扩展会回退到 REST API。
- 可以在设置页清除已保存的本地目录配置，之后重新选择目录或继续使用 REST API。
- Firefox 当前不支持本地目录句柄写入，仍使用 REST API 路径。

### 2. Reading Mode

Reading Mode 可设置阅读导出方式与高亮主题。存储内容高亮与 Obsidian 插件 [Sidebar Highlights](https://github.com/trevware/obsidian-sidebar-highlights) 配合使用更佳。

### 3. 路径模板配置

#### 文章路径模板

用于普通网页文章的保存路径，默认：`Clippings/{domain}/{yyyy}/{slug}.md`

**可用变量：**

- `{domain}` - 网站域名（支持映射）
- `{yyyy}` - 年份（4位）
- `{mm}` - 月份（2位）
- `{dd}` - 日期（2位）
- `{slug}` - 标题的 URL 友好格式
- `{title}` - 原始标题

**示例：**

- `Clippings/{domain}/{yyyy}/{slug}.md` → `Clippings/公众号/2024/my-article.md`
- `Articles/{yyyy}-{mm}/{domain}/{title}.md` → `Articles/2024-01/知乎/我的文章.md`

#### AI 对话路径模板

用于 AI 对话的保存路径，默认：`AI/{platform}/{yyyy}/{yyyy}-{mm}-{dd}_{title}.md`

**可用变量：**

- `{platform}` - AI 平台名称（如 chatgpt, claude）
- `{yyyy}` - 年份
- `{mm}` - 月份
- `{dd}` - 日期
- `{title}` - 对话标题

### 4. 🌐 域名映射（新功能）

将网站域名映射为更友好的名称，让你的文件夹结构更清晰。

#### 默认映射

系统预设了常用网站的映射：

| 域名                | 映射名称 |
| ------------------- | -------- |
| mp.weixin.qq.com    | 公众号   |
| wx.zsxq.com         | 知识星球 |
| www.zhihu.com       | 知乎     |
| juejin.cn           | 掘金     |
| www.youtube.com     | YouTube  |
| twitter.com / x.com | Twitter  |
| medium.com          | Medium   |

#### 自定义映射

你可以：

- ✏️ **编辑现有映射**：直接修改域名或名称
- ➕ **添加新映射**：点击"+ 添加映射"按钮
- 🗑️ **删除映射**：点击每行的"删除"按钮

**使用场景：**

- 将 `blog.example.com` 映射为 `我的博客`
- 将 `docs.company.com` 映射为 `公司文档`
- 将长域名映射为简短易记的名称

#### 映射效果示例

**不使用映射：**

```
Clippings/mp.weixin.qq.com/2024/article-title.md
```

**使用映射后：**

```
Clippings/公众号/2024/article-title.md
```

### 5. 🤖 AI 分类器

使用 AI 自动分析内容并添加标签和分类。

#### 配置选项

- **启用开关**：勾选以启用 AI 分类功能
- **Provider**：选择 AI 提供商
  - Ollama (本地)
  - OpenAI
  - 兼容接口
- **Endpoint**：API 端点地址
- **Model**：使用的模型名称
- **API Key**：如果需要的话
- **Taxonomy**：分类体系（JSON 格式）

#### Taxonomy 示例

```json
{
  "type": ["article", "ai_chat"],
  "topics": ["cs", "math", "product", "research", "howto", "news", "misc"],
  "ai_platform": ["chatgpt", "claude", "gemini", "copilot", "perplexity", "poe", "other"]
}
```

### 6. 🔍 配置诊断

点击"🔍 诊断配置"按钮可以：

- 查看当前配置详情
- 检查配置问题
- 测试 API 连接
- 自动修复常见问题

## 💡 使用技巧

### 1. 组织文件结构

结合域名映射和路径模板，创建清晰的文件结构：

```
Clippings/
├── 公众号/
│   └── 2024/
│       ├── article-1.md
│       └── article-2.md
├── 知乎/
│   └── 2024/
│       └── question-answer.md
└── 知识星球/
    └── 2024/
        └── discussion.md
```

### 2. 按时间归档

使用年月日变量创建时间归档：

```
Clippings/{yyyy}/{mm}/{domain}/{slug}.md
```

结果：

```
Clippings/2024/01/公众号/my-article.md
```

### 3. 混合使用

结合多种变量创建复杂的路径结构：

```
{domain}/{yyyy}-{mm}/{slug}.md
```

### 4. 快速测试

1. 修改配置后点击"💾 保存配置"
2. 点击"🔍 诊断配置"测试连接
3. 如有问题，点击"🔧 修复配置"

## 🔧 故障排除

### 配置未生效

- 确保点击了"保存配置"按钮
- 查看是否有错误提示
- 使用"诊断配置"检查问题

### 连接失败

- 检查 Obsidian Local REST API 插件是否启用
- 确认 API Key 是否正确
- 尝试使用"修复配置"功能

### 本地目录不可用

- 确认正在使用 Chromium 浏览器
- 在设置页重新点击本地目录并允许浏览器权限提示
- 如果不想继续使用本地目录，清除目录配置并确认 REST API 可用

### 域名映射不工作

- 确保域名完全匹配（包括 www 前缀）
- 检查映射名称是否包含非法字符
- 保存配置后重新测试
