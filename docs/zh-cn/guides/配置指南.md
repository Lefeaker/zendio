# Zendio - 配置指南

## 📋 快速开始

### 1. 安装 Obsidian Local REST API 插件

1. 打开 Obsidian
2. 进入 `设置` → `第三方插件` → `浏览`
3. 搜索 "Local REST API"
4. 安装并启用插件

### 2. 配置 Local REST API 插件

1. 在 Obsidian 设置中找到 `Local REST API`
2. 记录以下信息：
   - **HTTPS 端口**（通常是 27124）
   - **HTTP 端口**（通常是 27123）
   - **API Key**（点击生成或复制现有的）
   - **Vault 名称**（你的笔记库名称）

### 3. 配置浏览器扩展

1. 右键点击扩展图标，选择 `选项`
2. 填写配置信息：

```
HTTPS URL: https://127.0.0.1:27124/
HTTP URL:  http://127.0.0.1:27123/
Vault:     your-vault-name
API Key:   your-api-key-here
```

3. 点击 `保存`

## 💡 配置说明

### HTTPS 和 HTTP URL

**为什么需要配置两个 URL？**

- 某些设备上 HTTPS 可能存在认证问题
- 配置两个 URL 后，扩展会自动选择可用的连接
- 提高连接成功率和稳定性

**如何填写？**

- **HTTPS URL**: `https://127.0.0.1:端口号/`
- **HTTP URL**: `http://127.0.0.1:端口号/`
- 端口号可以在 Obsidian 的 Local REST API 插件设置中查看
- 通常 HTTPS 端口为 27124，HTTP 端口为 27123

**示例**:

```
HTTPS URL: https://127.0.0.1:27124/
HTTP URL:  http://127.0.0.1:27123/
```

### Vault 名称

- 这是你的 Obsidian 笔记库的名称
- 可以在 Obsidian 左下角看到
- 或者在 `设置` → `关于` 中查看

### API Key

- 在 Local REST API 插件设置中生成或复制
- 这是一个长字符串，类似：`704187227d8368dd93a29d4e5ec64c45da5d571bcfa0cc48d6a09d24e2bafe7c`
- 请妥善保管，不要分享给他人

## 🎯 使用方法

### 剪藏网页

1. 在任意网页上右键点击
2. 选择 `剪藏到 Obsidian（后台）`
3. 等待通知提示成功

### 查看日志

1. 访问 `chrome://extensions/`
2. 找到扩展，点击 `Service Worker`
3. 在控制台中查看详细日志

## 🔍 故障排查

### 连接失败

**检查清单**:

1. ✅ Obsidian 是否正在运行？
2. ✅ Local REST API 插件是否已启用？
3. ✅ URL 和端口号是否正确？
4. ✅ API Key 是否正确？
5. ✅ Vault 名称是否正确？

**查看日志**:

- 打开扩展的 Service Worker 控制台
- 查看详细的连接尝试日志
- 根据错误信息调整配置

### HTTPS 连接问题

如果 HTTPS 连接失败：

1. 扩展会自动尝试 HTTP 连接
2. 无需手动干预
3. 可以在日志中看到切换过程

### 认证失败

如果提示认证失败：

1. 检查 API Key 是否正确
2. 确保没有多余的空格
3. 尝试重新生成 API Key

## 📝 模板配置

### 文章路径模板

默认: `Clippings/{domain}/{yyyy}/{slug}.md`

**可用变量**:

- `{domain}`: 网站域名
- `{yyyy}`: 年份（4位）
- `{mm}`: 月份（2位）
- `{dd}`: 日期（2位）
- `{slug}`: 文章标题（URL 友好格式）
- `{title}`: 文章标题（原始格式）

**示例**:

```
Clippings/{domain}/{yyyy}/{slug}.md
→ Clippings/example.com/2025/my-article.md

Clippings/{yyyy}-{mm}-{dd}/{title}.md
→ Clippings/2025-09-30/My Article.md
```

### AI 对话路径模板

默认: `AI/{platform}/{yyyy}/{yyyy}-{mm}-{dd}_{title}.md`

**可用变量**:

- `{platform}`: AI 平台名称（chatgpt, claude 等）
- `{yyyy}`, `{mm}`, `{dd}`: 日期
- `{title}`: 对话标题

## 🚀 高级功能

### AI 分类器

可以启用 AI 分类器自动分类内容：

1. 勾选 `启用分类器`
2. 选择 Provider（ollama / openai / compatible）
3. 配置 Endpoint 和 Model
4. 自定义 Taxonomy（分类体系）

**注意**: 这是可选功能，不启用也可以正常使用剪藏功能。
