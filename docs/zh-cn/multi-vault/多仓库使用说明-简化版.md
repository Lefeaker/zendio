# 多仓库功能使用说明（简化版）

## 🎯 功能概述

现在你可以配置多个 Obsidian 仓库，并通过路由规则自动将内容保存到不同的仓库！

### 工作原理

```
剪藏内容
    ↓
检查路由规则
    ↓
匹配成功？
├─ 是 → 保存到对应仓库
└─ 否 → 保存到默认仓库
```

## 📋 配置步骤

### 1. 配置默认仓库

在设置页面的 **"Obsidian Local REST API"** 部分：

- 这是你的**默认仓库**
- 不符合任何路由规则的内容都会保存到这里
- 配置方式和之前一样

### 2. 添加额外仓库

在 **"额外仓库"** 部分：

1. 点击 **"+ 添加仓库"** 按钮
2. 会自动添加一个新的仓库表单，包含以下字段：
   - **仓库名称**：修改为你的仓库名称（如"工作笔记"）
   - **HTTPS URL**：修改为你的 HTTPS 地址
   - **HTTP URL**：修改为你的 HTTP 地址
   - **Vault 名称**：修改为你的 Obsidian vault 名称
   - **API Key**：输入你的 API 密钥

3. 直接在表单中修改各个字段
4. 点击"删除"按钮可以删除仓库

### 3. 添加路由规则

在 **"路由规则"** 部分：

1. 点击 **"+ 添加规则"** 按钮
2. 会自动添加一个新的规则表单，包含以下字段：
   - **规则类型**：从下拉菜单选择
     - 域名匹配：根据网站域名路由
     - 关键词匹配：根据标题和内容中的关键词路由
     - URL 模式：使用正则表达式匹配 URL
   - **匹配模式**：输入匹配规则
     - 域名：`github.com` 或 `*.google.com`
     - 关键词：`编程,代码,开发`（用逗号分隔）
     - URL 模式：`^https://github\.com/.*/issues/\d+$`
   - **目标仓库**：从下拉菜单选择
   - **优先级**：输入 0-100 的数字（数字越大优先级越高）
   - **描述**：可选，帮助你记住这个规则的用途
   - **启用此规则**：勾选框控制是否启用

3. 直接在表单中修改各个字段
4. 点击"删除"按钮可以删除规则

### 4. 保存配置

点击页面底部的 **"保存配置"** 按钮。

### 5. 重新加载扩展

```
1. 打开 chrome://extensions/
2. 找到 "Zendio" 扩展
3. 点击 "重新加载" 按钮 🔄
```

## 💡 使用示例

### 示例 1：工作与生活分离

**配置：**

- 默认仓库：个人笔记
- 额外仓库：工作笔记

**路由规则：**

```
规则 1：
  类型：域名匹配
  模式：github.com
  目标：工作笔记
  优先级：20
  描述：GitHub 内容保存到工作仓库

规则 2：
  类型：域名匹配
  模式：stackoverflow.com
  目标：工作笔记
  优先级：20
  描述：Stack Overflow 保存到工作仓库
```

**效果：**

- 访问 github.com 剪藏 → 保存到工作笔记
- 访问 stackoverflow.com 剪藏 → 保存到工作笔记
- 访问其他网站剪藏 → 保存到个人笔记（默认）

### 示例 2：按主题分类

**配置：**

- 默认仓库：杂项笔记
- 额外仓库 1：技术笔记
- 额外仓库 2：产品笔记

**路由规则：**

```
规则 1：
  类型：关键词匹配
  模式：编程,代码,开发,技术,算法
  目标：技术笔记
  优先级：30
  描述：技术相关内容

规则 2：
  类型：关键词匹配
  模式：产品,设计,用户体验,UI,UX
  目标：产品笔记
  优先级：30
  描述：产品设计相关内容
```

**效果：**

- 标题包含"编程"的文章 → 技术笔记
- 内容包含"产品设计"的文章 → 产品笔记
- 其他内容 → 杂项笔记（默认）

### 示例 3：特定网站的特定页面

**配置：**

- 默认仓库：通用笔记
- 额外仓库：GitHub Issues

**路由规则：**

```
规则 1：
  类型：URL 模式
  模式：^https://github\.com/.*/issues/\d+$
  目标：GitHub Issues
  优先级：50
  描述：只匹配 GitHub Issues 页面
```

**效果：**

- `https://github.com/user/repo/issues/123` → GitHub Issues
- `https://github.com/user/repo` → 通用笔记（默认）

## 🎯 规则匹配说明

### 域名匹配

**精确匹配：**

- `github.com` - 只匹配 github.com
- `www.example.com` - 只匹配 www.example.com

**通配符匹配：**

- `*.github.com` - 匹配所有 github.com 的子域名
- `*.google.com` - 匹配 mail.google.com, drive.google.com 等

### 关键词匹配

- 多个关键词用**逗号**分隔
- 任意一个关键词匹配即可
- **不区分大小写**
- 在标题和内容前 2000 字符中搜索

### URL 模式

使用正则表达式匹配完整 URL。

**常用模式：**

```
GitHub Issues:  ^https://github\.com/.*/issues/\d+$
Medium 文章:    ^https://medium\.com/@.+/.+$
YouTube 视频:   ^https://www\.youtube\.com/watch\?v=.+$
```

## 📊 优先级说明

- 规则按**优先级从高到低**匹配
- 找到第一个匹配的规则后立即使用
- 建议优先级分配：
  - **50-100**：特定规则（如特定 URL 模式）
  - **10-49**：一般规则（如域名、关键词）
  - **0-9**：兜底规则（如通配符域名）

## 🔍 查看路由日志

剪藏后，打开浏览器控制台（F12）查看详细日志：

```
[VaultRouter] Selected vault: 工作笔记
[VaultRouter] Matched rule: GitHub 内容保存到工作仓库 -> 工作笔记
```

通知消息也会显示目标仓库：

```
已保存到 Obsidian (工作笔记)
```

## ⚠️ 注意事项

1. **修改规则后需要重新加载扩展**才能生效
2. **删除仓库会同时删除相关的路由规则**
3. **至少保留一个仓库**（默认仓库）
4. **规则数量建议不超过 20 条**，过多会影响性能
5. **关键词匹配只搜索前 2000 字符**，超长内容可能匹配不准确

## 🐛 故障排查

### 问题 1：内容保存到错误的仓库

**解决方法：**

1. 打开浏览器控制台（F12）
2. 查看 `[VaultRouter]` 开头的日志
3. 确认哪个规则被匹配
4. 调整规则的优先级或匹配模式

### 问题 2：规则不生效

**检查清单：**

- [ ] 是否点击了"保存配置"按钮
- [ ] 是否重新加载了扩展
- [ ] 匹配模式是否正确（检查拼写）
- [ ] 目标仓库是否存在

### 问题 3：无法连接到仓库

**检查清单：**

- [ ] Obsidian 正在运行
- [ ] Local REST API 插件已启用
- [ ] API Key 正确
- [ ] HTTPS/HTTP URL 正确
- [ ] 防火墙未阻止连接

## 💡 最佳实践

1. **从简单开始**：先添加 1-2 个规则，测试通过后再添加更多
2. **清晰的描述**：为每个规则添加描述，方便日后管理
3. **合理的优先级**：特定规则优先级高，通用规则优先级低
4. **定期清理**：删除不再使用的仓库和规则
5. **测试验证**：添加规则后立即测试，确保符合预期
