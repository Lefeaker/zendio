# 多仓库功能快速开始

## 🎉 新功能：多仓库智能路由

Zendio 现在支持配置多个 Obsidian 仓库，并通过智能路由规则自动选择目标仓库！

## 🚀 5 分钟快速上手

### 步骤 1：重新加载扩展

```
1. 打开 chrome://extensions/
2. 找到 "Zendio" 扩展
3. 点击 "重新加载" 按钮 🔄
```

### 步骤 2：打开多仓库管理

```
1. 右键点击扩展图标 → 选择 "选项"
2. 在设置页面找到 "多仓库管理" 部分
3. 点击 "🗂️ 打开多仓库管理" 按钮
```

### 步骤 3：添加第一个仓库

系统会自动创建一个默认仓库。你可以：

**选项 A：编辑默认仓库**

1. 点击默认仓库旁边的 "编辑" 按钮
2. 修改仓库名称（如"个人笔记"）
3. 填写正确的配置信息
4. 点击 "保存"

**选项 B：添加新仓库**

1. 点击 "+ 添加仓库" 按钮
2. 填写仓库信息：
   - **仓库名称**：如"工作笔记"
   - **HTTPS URL**：`https://127.0.0.1:27124/`
   - **HTTP URL**：`http://127.0.0.1:27123/`
   - **Vault**：Obsidian 中的 vault 名称
   - **API Key**：从 Obsidian Local REST API 插件获取
3. 点击 "保存"

### 步骤 4：添加路由规则（可选）

如果你只有一个仓库，可以跳过此步骤。如果有多个仓库，建议添加路由规则：

**示例 1：按域名路由**

```
规则类型：域名匹配
匹配模式：github.com
目标仓库：工作笔记
优先级：10
描述：GitHub 内容保存到工作仓库
```

**示例 2：按关键词路由**

```
规则类型：关键词匹配
匹配模式：旅行,美食,电影
目标仓库：个人笔记
优先级：10
描述：生活相关内容保存到个人仓库
```

### 步骤 5：测试剪藏

1. 打开一个网页（如 https://github.com）
2. 点击扩展图标或右键选择剪藏
3. 查看通知消息，确认内容保存到正确的仓库

## 🎯 规则匹配说明

### 域名匹配

**精确匹配：**

- `github.com` - 只匹配 github.com
- `www.example.com` - 只匹配 www.example.com

**通配符匹配：**

- `*.github.com` - 匹配所有 github.com 的子域名
- `*.google.com` - 匹配 mail.google.com, drive.google.com 等

### 关键词匹配

- 多个关键词用**逗号**分隔
- 任意一个关键词匹配即可
- **不区分大小写**
- 在标题和内容中搜索

**示例：**

```
模式：Python,JavaScript,编程
匹配：标题包含 "Python 入门教程" ✅
匹配：内容包含 "JavaScript 代码" ✅
不匹配：标题和内容都不包含关键词 ❌
```

### URL 模式

使用正则表达式匹配完整 URL。

**示例：**

```
模式：^https://github\.com/.*/issues/\d+$
匹配：https://github.com/user/repo/issues/123 ✅
不匹配：https://github.com/user/repo ❌
```

## 💡 最佳实践

### 1. 规则优先级设置

- **特定规则**：50-100（如特定域名）
- **一般规则**：10-49（如关键词匹配）
- **兜底规则**：0-9（如通配符域名）

### 2. 规则数量建议

- 建议不超过 **20 条**规则
- 过多规则会影响性能
- 定期清理不需要的规则

### 3. 测试建议

添加新规则后：

1. 打开符合规则的网页
2. 执行剪藏操作
3. 检查通知消息中的仓库名称
4. 打开浏览器控制台查看详细日志

### 4. 规则描述

为每个规则添加清晰的描述：

- ✅ 好的描述："GitHub Issues 保存到工作仓库"
- ❌ 不好的描述："规则1"

## 🔧 故障排查

### 问题 1：内容保存到错误的仓库

**检查步骤：**

1. 打开浏览器控制台（F12）
2. 查看 `[VaultRouter]` 开头的日志
3. 确认哪个规则被匹配
4. 调整规则的优先级或匹配模式

### 问题 2：规则不生效

**可能原因：**

- 规则未启用（检查"启用"复选框）
- 匹配模式错误（检查拼写）
- 优先级太低（被其他规则覆盖）

**解决方法：**

1. 确认规则已启用
2. 测试匹配模式是否正确
3. 提高规则优先级

### 问题 3：无法连接到仓库

**检查清单：**

- [ ] Obsidian 正在运行
- [ ] Local REST API 插件已启用
- [ ] API Key 正确
- [ ] HTTPS/HTTP URL 正确
- [ ] 防火墙未阻止连接

## 🎓 进阶技巧

### 技巧 1：使用正则表达式

匹配特定 URL 模式：

```
# 匹配所有 GitHub Issues
^https://github\.com/.*/issues/\d+$

# 匹配所有 Medium 文章
^https://medium\.com/@.+/.+$

# 匹配所有 YouTube 视频
^https://www\.youtube\.com/watch\?v=.+$
```

### 技巧 2：组合多个规则

为同一个仓库创建多个规则：

```
规则 1：域名匹配 github.com → 工作笔记 (优先级 20)
规则 2：域名匹配 gitlab.com → 工作笔记 (优先级 20)
规则 3：关键词匹配 代码,编程 → 工作笔记 (优先级 15)
```

### 技巧 3：临时禁用规则

不想删除规则但暂时不用？

1. 编辑规则
2. 取消勾选"启用此规则"
3. 保存
