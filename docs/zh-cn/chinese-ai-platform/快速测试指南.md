# Gemini 图片捕捉 - 快速测试指南

## 🚀 快速开始（5分钟）

### 1️⃣ 重新加载扩展
代码已经编译完成，在浏览器中重新加载扩展即可。

### 2️⃣ 测试步骤
1. 打开 https://gemini.google.com/
2. 上传一张图片（任意图片）
3. 等待 Gemini 回复
4. 按 F12 打开开发者工具
5. 切换到 **Console** 标签
6. 点击扩展图标进行剪藏

### 3️⃣ 查看结果

#### 在控制台中查找这些信息：

```
[Gemini] Found X img elements in light DOM
```
- 如果 X = 0：Gemini 没有使用标准 img 标签
- 如果 X > 0：继续查看详细信息

```
[Gemini] Image 1: src="https://..." 
```
- ✅ 如果是 `https://` 开头：**可以捕捉**
- ❌ 如果是 `blob:` 开头：**无法捕捉**（会显示 "BLOB URL - will be skipped"）

```
[Gemini] Found X images in Shadow DOM
```
- 如果有这条消息：图片在 Shadow DOM 中，当前版本可能无法捕捉

#### 检查生成的 Markdown

在 Obsidian 中查看剪藏的文件，搜索 `![`：

**成功**：
```markdown
![Image description](https://lh3.googleusercontent.com/...)
```

**失败**：
- 没有找到 `![` 标记
- 或者是空的：`![]()`

## 📊 三种可能的结果

### ✅ 结果 1：成功捕捉
**控制台显示**：
```
[Gemini] Image 1: src="https://lh3.googleusercontent.com/..."
```

**Markdown 中**：
```markdown
![Uploaded image](https://lh3.googleusercontent.com/...)
```

**结论**：问题已解决！Gemini 使用永久 HTTPS URLs。

---

### ❌ 结果 2：只有 Blob URLs
**控制台显示**：
```
[Gemini] Image 1: src="blob:https://gemini.google.com/xxx" (BLOB URL - will be skipped)
```

**Markdown 中**：
没有图片标记

**结论**：Gemini 只使用临时 Blob URLs，需要其他解决方案。

**可能的解决方案**：
1. 手动保存图片到本地
2. 使用截图功能
3. 等待 Gemini 更新（如果他们将来提供永久链接）

---

### ⚠️ 结果 3：图片在 Shadow DOM
**控制台显示**：
```
[Gemini] Found 0 img elements in light DOM
[Gemini] Found 2 images in Shadow DOM of <image-query>
```

**结论**：图片封装在 Shadow DOM 中，需要额外的代码来访问。

**下一步**：需要修改代码来访问 Shadow DOM（技术上可行）。

## 🔍 详细诊断（如果需要）

### 手动检查图片元素

1. 在 Gemini 页面中，右键点击上传的图片
2. 选择 "检查元素"（Inspect）
3. 在开发者工具中查看图片的 HTML 结构

**示例 1：标准 img 标签**
```html
<img src="https://lh3.googleusercontent.com/..." alt="Uploaded image">
```
→ 应该可以捕捉

**示例 2：Blob URL**
```html
<img src="blob:https://gemini.google.com/abc-123-def">
```
→ 无法捕捉（临时链接）

**示例 3：Shadow DOM**
```html
<image-query>
  #shadow-root (open)
    <img src="https://...">
</image-query>
```
→ 需要特殊处理

### 检查所有属性

在控制台运行：
```javascript
const img = document.querySelector('img');
if (img) {
  console.log('All attributes:');
  for (let attr of img.attributes) {
    console.log(`  ${attr.name}: ${attr.value}`);
  }
}
```

查找是否有其他属性存储了图片 URL：
- `data-src`
- `data-original-src`
- `data-image-url`
- `data-url`
