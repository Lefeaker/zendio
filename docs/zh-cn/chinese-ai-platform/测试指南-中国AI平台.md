# 中国AI平台测试指南

## 🎯 测试目标

验证插件对通义千问、DeepSeek、Kimi 三个中国AI平台的支持是否正常工作。

## 📋 前置准备

### 1. 安装扩展

```bash
cd AiiinOB/your-extension
npm install
npm run build
```

### 2. 加载扩展到浏览器

1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 开启"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择 `dist` 文件夹

### 3. 配置 Obsidian

1. 确保 Obsidian 正在运行
2. 确保 Local REST API 插件已启用
3. 记录 API 地址和密钥

### 4. 配置扩展

1. 右键扩展图标 → 选项
2. 填入 Obsidian REST API 配置
3. 保存设置

## 🧪 测试步骤

### 测试 1：通义千问

#### 步骤

1. **访问网站**
   - 打开 https://tongyi.aliyun.com
   - 登录你的阿里云账号

2. **创建对话**
   - 点击"新建会话"
   - 输入测试问题：
     ```
     请用Python写一个快速排序算法，并解释其时间复杂度
     ```
   - 等待AI回复

3. **测试剪藏**
   - 打开浏览器控制台（F12）
   - 点击扩展图标
   - 观察控制台输出

4. **验证结果**
   - 检查控制台是否显示：
     ```
     Is AI chat: true
     platform: tongyi
     ```
   - 在 Obsidian 中查看导出的文件
   - 验证文件路径：`AI/tongyi/2025/...`

#### 预期结果

- ✅ 页面被识别为 AI 聊天
- ✅ 平台识别为 tongyi
- ✅ 提取到用户和助手的消息
- ✅ 代码块格式正确
- ✅ 文件保存到正确路径

#### 可能的问题

- ❌ 如果显示 `Is AI chat: false`
  - 检查 URL 是否包含 `tongyi.aliyun.com`
  - 确认已重新编译并重新加载扩展
- ❌ 如果没有提取到消息
  - 检查页面是否完全加载
  - 查看控制台是否有错误信息
  - 页面结构可能已更新，需要调整选择器

## 🐛 问题排查

### 问题 1：页面未被识别为 AI 聊天

**症状**：控制台显示 `Is AI chat: false`

**解决方案**：

1. 检查 URL 是否正确
2. 确认 `src/content/detect.ts` 已更新
3. 重新编译：`npm run build`
4. 重新加载扩展

### 问题 2：没有提取到消息

**症状**：导出的文件中没有对话内容

**解决方案**：

1. 检查页面是否完全加载
2. 查看控制台错误信息
3. 检查选择器是否匹配页面结构
4. 尝试刷新页面后再剪藏

### 问题 3：角色识别错误

**症状**：用户和助手的消息角色混淆

**解决方案**：

1. 检查页面的 DOM 结构
2. 更新 `parse.ts` 中的角色识别逻辑
3. 添加更多的识别条件

### 问题 4：格式混乱

**症状**：代码块、列表等格式不正确

**解决方案**：

1. 检查 HTML 到 Markdown 的转换
2. 查看 `chatHtmlToMarkdown` 函数
3. 可能需要添加特殊处理

### 问题 5：模型信息缺失

**症状**：frontmatter 中没有 model 字段

**解决方案**：

1. 检查页面上是否显示模型信息
2. 更新模型识别的正则表达式
3. 添加更多的查找位置
