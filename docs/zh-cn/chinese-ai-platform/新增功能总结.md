## 🔧 技术实现

### 1. 平台检测
```typescript
// src/content/detect.ts
const AI_CHAT_URL_PATTERNS = [
  /(chatgpt\.com|chat\.openai\.com)/i,
  /claude\.ai/i,
  /gemini\.google\.com/i,
  /kimi\.(moonshot\.cn|com)/i,
  /deepseek\.com/i,
  /tongyi\.(aliyun\.com|com)/i
];

export const isAIChat = (url: string) => {
  return AI_CHAT_URL_PATTERNS.some((pattern) => pattern.test(url));
};
```

### 2. DOM 解析
每个平台都有专门的提取函数：
- `extractTongyiChatData(doc: Document): ParsedResult`
- `extractDeepSeekChatData(doc: Document): ParsedResult`
- `extractKimiChatData(doc: Document): ParsedResult`

### 3. 通用策略
所有平台都使用了相似的实现策略：

#### 多选择器策略
```typescript
const possibleSelectors = [
  '[class*="message-item"]',
  '[class*="messageItem"]',
  '[class*="chat-message"]',
  '[class*="chatMessage"]',
  '[class*="Message"]',
  '[data-role="user"], [data-role="assistant"]',
  '[data-type="user"], [data-type="assistant"]',
  'article',
];
```

#### 多层次角色识别
1. 检查 data-role 和 data-type 属性
2. 检查 CSS 类名中的关键词
3. 查找子元素中的角色指示器
4. 默认为 assistant

#### 智能模型识别
1. 从按钮文本中提取
2. 从模型选择器中提取
3. 从页面元素中搜索
4. 支持多种命名格式

## 🎨 用户体验

### 使用流程
1. 访问支持的 AI 平台（如 tongyi.aliyun.com）
2. 进行对话
3. 点击浏览器扩展图标
4. 选择"剪藏到 Obsidian"
5. 在 Obsidian 中查看导出的 Markdown 文件

### 导出格式
```markdown
---
title: 对话标题
platform: tongyi/deepseek/kimi
model: 模型名称
url: 原始URL
messageCount: 消息数量
clippedAt: 导出时间
---

# 对话标题

## USER
用户的问题...

## ASSISTANT
AI的回答...
```

## 🐛 已知限制

### 当前限制
1. **页面结构依赖**：依赖于平台的 DOM 结构，如果平台更新 UI 可能需要调整
2. **动态加载**：某些内容可能需要滚动加载，需要等待页面完全加载
3. **图片处理**：目前保留外链，不下载到本地

### 潜在问题
1. **选择器失效**：平台更新可能导致选择器失效
2. **特殊格式**：某些特殊格式可能需要额外处理
3. **性能优化**：长对话的处理性能可能需要优化
