# 通义千问支持说明

## 概述

已成功添加对通义千问（Tongyi Qianwen）的支持。现在插件可以识别并导出通义千问的对话内容。

## 支持的平台

目前插件支持以下 AI 平台：

1. **ChatGPT** - `chatgpt.com`, `chat.openai.com`
2. **Claude** - `claude.ai`
3. **Gemini** - `gemini.google.com`
4. **Copilot** - `copilot.microsoft.com`
5. **通义千问** - `tongyi.aliyun.com` ✨ 新增

## 实现细节

### 1. 平台检测

在 `src/content/adapters/chat.ts` 中添加了通义千问的URL检测：

```typescript
function detectPlatform(url: string): string {
  if (/chatgpt/.test(url)) return 'chatgpt';
  if (/claude/.test(url))  return 'claude';
  if (/gemini/.test(url))  return 'gemini';
  if (/copilot/.test(url)) return 'copilot';
  if (/tongyi/.test(url)) return 'tongyi';  // 新增
  if (/perplexity/.test(url)) return 'perplexity';
  return 'chat';
}
```

### 2. DOM 解析

在 `src/third_party/ai-chat-exporter/parse.ts` 中添加了 `extractTongyiChatData` 函数，用于解析通义千问的对话结构。

#### 选择器策略

由于通义千问的页面结构可能会变化，实现了多种选择器策略：

```typescript
const possibleSelectors = [
  '[class*="message-item"]',
  '[class*="messageItem"]',
  '[class*="chat-message"]',
  '[class*="chatMessage"]',
  '[data-role="user"], [data-role="assistant"]',
  '[data-type="user"], [data-type="assistant"]',
];
```

#### 角色识别

通过多种方式识别消息的角色（用户/助手）：

1. 检查 `data-role` 属性
2. 检查 `data-type` 属性
3. 检查 CSS 类名中的关键词（user, assistant, bot）
4. 查找子元素中的角色指示器

#### 模型识别

尝试从页面中提取模型信息：

- 匹配模式：`通义千问`, `Qwen`, `qwen-max`, `qwen-plus`, `qwen-turbo`
- 查找位置：按钮、模型选择器等元素

### 3. 标题提取

从文档标题中提取对话标题，并移除平台相关的后缀：

- 移除 " - 通义"
- 移除 " - 通义千问"

## 使用方法

1. 访问通义千问网站：https://tongyi.aliyun.com
2. 进行对话
3. 点击浏览器扩展图标
4. 选择"剪藏到 Obsidian"
5. 对话内容将以 Markdown 格式保存到 Obsidian

## 导出格式

导出的 Markdown 文件包含：

- 对话标题
- 平台信息（tongyi）
- 模型信息（如果可用）
- 完整的对话历史（用户和助手的消息）
- 元数据（URL、消息数量、导出时间等）

## 测试建议

1. **基本对话测试**
   - 创建一个简单的对话

   - 验证消息角色识别是否正确

2. **复杂内容测试**
   - 包含代码块的对话

   - 包含表格的对话

3. **模型识别测试**
   - 测试不同模型（qwen-max, qwen-plus, qwen-turbo）

4. **标题测试**
   - 验证对话标题是否正确提取
   - 测试中文和英文标题

## 注意事项

1. **页面结构变化**：通义千问的页面结构可能会更新，如果导出失败，可能需要更新选择器。

2. **登录状态**：确保已登录通义千问账号，否则可能无法访问完整的对话历史。

3. **动态加载**：如果对话内容是动态加载的，请确保页面完全加载后再进行导出。

## 后续优化
