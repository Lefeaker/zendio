# All-in-Obsidian 权限使用说明

公开访问版本：https://github.com/Lefeaker/AllinOB

最后更新：2025-02-14

本文档用于 Chrome Web Store 审核与用户透明披露，说明 All-in-Obsidian Chrome 扩展（版本 0.1.0）在 `manifest.json` 中声明的权限及其使用场景。所有权限仅在用户触发相关功能时生效，并遵循最小授权原则。

## Host 权限

- **声明范围**：`<all_urls>`, `http://127.0.0.1/*`, `https://127.0.0.1/*`
- **使用场景**：
  - 在任意网页上执行剪藏、批注、AI 对话归档等功能。
  - 当用户发起写入时，与本地运行的 Obsidian REST API（默认监听在 `127.0.0.1`）通信，将整理后的 Markdown 写入 Vault。
- **限制措施**：
  - 仅在用户主动操作（点击扩展按钮、快捷键或右键菜单）后访问当前活动标签页，不在后台巡检网页。
  - 对本地网络仅访问用户配置的回环地址，不扫描其他主机。

## `activeTab`

- **使用场景**：读取当前激活标签页的 DOM、选中文本与元信息，用于生成剪藏内容与元数据。
- **限制措施**：作用范围限定在当前标签页，未激活的页面不会被访问或存储。

## `scripting`

- **使用场景**：注入前端脚本以呈现浮动剪藏面板、捕获选中内容、解析 AI 对话结构。
- **限制措施**：脚本随扩展打包发布，可审计；仅在用户交互后注入，不在后台循环运行。

## `storage`

- **使用场景**：保存扩展配置（Obsidian API 设置、模板、路由规则、语言偏好等）以及少量运行状态。
- **限制措施**：数据仅保存在浏览器本地的扩展存储空间，不会同步或上传到任何第三方服务。

## `contextMenus`

- **使用场景**：在网页右键菜单中提供“保存到 Obsidian”等入口，帮助用户快捷触发剪藏。
- **限制措施**：菜单仅作为浏览器 UI 功能，不会主动修改网页内容。

## `notifications`

- **使用场景**：在剪藏成功、写入失败或需要提醒时，通过桌面通知反馈结果。
- **限制措施**：通知内容仅包含状态提示，不展示剪藏正文或敏感数据，且不会触发外部网络请求。
