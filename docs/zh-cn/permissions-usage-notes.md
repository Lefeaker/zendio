# All-in-Obsidian Chrome 扩展权限使用说明

最后更新：2025-02-14

本说明用于 Chrome Web Store 审核问卷与权限面板填写，帮助审核人员理解扩展声明的每项权限及其限制。所有权限均遵循最小授权原则，仅在必要时启用。

## 1. Host 权限

- **申请范围**：`https://*/*`, `http://*/*`, `https://127.0.0.1/*`
- **使用场景**：
  - 对用户主动打开的网页执行剪藏、内容提取与格式化。
  - 在用户触发写入操作时，将处理后的内容通过 Obsidian REST API 写入本地 Vault（通常位于 `http://127.0.0.1:<port>`）。
- **限制措施**：
  - 扩展仅在用户明确操作（点击扩展图标、快捷键或右键菜单）时访问当前活动标签页。
  - 不会在后台批量抓取、监控或存储用户浏览记录。
  - 对本地网络仅访问用户配置的 Obsidian 服务地址，不扫描其他主机。

## 2. `storage`

- **使用场景**：保存扩展配置（如输出目录、模板、API Token、快捷键设置）及有限的用户偏好（如 UI 语言、功能开关）。
- **限制措施**：
  - 数据保存在 Chrome 扩展专用存储空间，未与第三方共享。
  - 存储内容最小化，仅保留维持功能所必需的键值。

## 3. `scripting`

- **使用场景**：允许扩展在用户指定的标签页执行内容提取脚本（例如解析网页结构、读取选中文本）。
- **限制措施**：
  - 脚本仅在用户发起剪藏时注入，不在后台持续运行。
  - 不会收集与剪藏无关的页面元素或用户输入。

## 4. `notifications`

- **使用场景**：在剪藏完成、同步失败或需要用户确认时，通过桌面通知给予即时反馈。
- **限制措施**：
  - 通知频率受控，仅在关键操作（成功/失败）后发送。
  - 通知内容仅包含操作状态，不展示剪藏正文或敏感信息。

## 5. 其他扩展可选权限（如适用）

- 若未来新增 `clipboardWrite` 或 `downloads` 权限，将仅用于实现显式新增的导出或复制功能，并在上线前更新本说明。
- 任何新增权限都会通过发布说明、隐私政策与扩展更新提示告知用户，并获取显式授权。
