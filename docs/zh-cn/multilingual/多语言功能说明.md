# 多语言功能说明

## 功能概述

All in Obsidian 现已支持多语言界面，用户可以在设置页面中选择自己喜欢的语言。

### 支持的语言

- 🇨🇳 **简体中文** (zh-CN) - 默认语言
- 🇺🇸 **English** (en)
- 🇯🇵 **日本語** (ja)

## 使用方法

### 切换语言

1. 点击浏览器工具栏中的扩展图标，或右键选择"设置"
2. 在设置页面顶部找到"语言设置"部分
3. 从下拉菜单中选择你喜欢的语言
4. 页面会自动刷新并应用新语言

### 多语言覆盖范围

多语言支持覆盖以下所有界面元素：

#### 1. 设置页面
- ✅ 所有标题和说明文字
- ✅ 表单标签和提示信息
- ✅ 按钮文字
- ✅ 成功/错误消息

#### 2. 右键菜单
- ✅ "剪藏整个页面到 Obsidian"
- ✅ "✂️ 剪藏选中内容到 Obsidian"

#### 3. 剪藏对话框
- ✅ 对话框标题
- ✅ 评论输入框标签和占位符
- ✅ 取消和剪藏按钮

#### 4. 通知消息
- ✅ 成功通知："已保存到 Obsidian"
- ✅ 失败通知："剪藏失败"、"内容提取失败"等

## 技术实现

### 架构设计

```
src/i18n/
├── index.ts       # i18n 工具函数
└── locales.ts     # 语言定义文件
```

### 核心功能

#### 1. 语言存储
- 使用 `chrome.storage.sync` 存储用户选择的语言
- 跨设备同步语言设置

#### 2. 自动初始化
- 页面加载时自动读取用户语言设置
- 使用 `data-i18n` 属性标记需要翻译的元素
- 自动替换所有标记元素的文本内容

#### 3. 动态切换
- 无需刷新页面即可切换语言
- 所有界面元素实时更新

### 代码示例

#### HTML 中使用多语言

```html
<!-- 文本内容 -->
<h2 data-i18n="apiConfigTitle">Obsidian Local REST API</h2>

<!-- 占位符 -->
<input type="text" data-i18n-placeholder="vaultNamePlaceholder" />

<!-- 标题属性 -->
<button data-i18n-title="saveButton">保存</button>
```

#### TypeScript 中使用多语言

```typescript
import { getMessages } from '../i18n';

// 获取所有消息
const msgs = await getMessages();

// 使用消息
console.log(msgs.saveSuccess);
```

## 添加新语言

如果你想添加新的语言支持，请按以下步骤操作：

### 1. 在 `src/i18n/locales.ts` 中添加语言代码

```typescript
export type Language = 'zh-CN' | 'en' | 'ja' | 'your-lang';
```

### 2. 添加翻译内容

```typescript
export const messages: Record<Language, Messages> = {
  // ... 现有语言
  'your-lang': {
    extensionName: 'Your Translation',
    // ... 其他翻译
  }
};
```

### 3. 在设置页面添加语言选项

编辑 `src/options/index.html`：

```html
<select id="languageSelect">
  <option value="zh-CN">简体中文</option>
  <option value="en">English</option>
  <option value="ja">日本語</option>
  <option value="your-lang">Your Language</option>
</select>
```

### 4. 更新语言列表

编辑 `src/i18n/index.ts` 中的 `getAvailableLanguages()` 函数：

```typescript
export function getAvailableLanguages(): Array<{ code: Language; name: string }> {
  return [
    { code: 'zh-CN', name: '简体中文' },
    { code: 'en', name: 'English' },
    { code: 'ja', name: '日本語' },
    { code: 'your-lang', name: 'Your Language' },
  ];
}
```

## 翻译指南

### 翻译原则

1. **保持简洁**：界面文字应简洁明了
2. **保持一致**：相同概念使用相同术语
3. **符合习惯**：使用目标语言的自然表达方式
4. **保留格式**：保留 emoji 和特殊符号

### 需要翻译的内容类型

#### 1. 标题和标签
- 简短、描述性
- 首字母大写（英文）

#### 2. 提示信息
- 以 💡 开头的提示
- 提供额外的上下文信息

#### 3. 按钮文字
- 动词开头
- 清晰表达操作

#### 4. 错误消息
- 以 ❌ 开头
- 说明问题和可能的解决方案

#### 5. 成功消息
- 以 ✅ 开头
- 确认操作已完成

### 常见问题

#### Q: 切换语言后部分文字没有更新？
A: 检查该元素是否添加了 `data-i18n` 属性。动态生成的元素需要在代码中手动设置文本。

#### Q: 如何测试特定语言？
A: 在设置页面选择对应语言，或在浏览器控制台执行：
```javascript
chrome.storage.sync.set({ language: 'en' });
```

#### Q: 语言设置会同步到其他设备吗？
A: 是的，使用 `chrome.storage.sync` 存储，会自动同步到登录同一账号的所有设备。

## 技术细节

### 性能优化

- 语言文件在构建时打包，无需额外加载
- 使用 `chrome.storage.sync` 缓存语言设置
- 初始化时批量更新 DOM，避免多次重绘

### 兼容性

- 支持 Chrome 88+
- 支持 Edge 88+
- 支持 Firefox 89+（需要 polyfill）

### 文件大小

- 每种语言约 2-3 KB
- 总体增加约 10 KB（未压缩）
- 对扩展性能影响可忽略不计
