# Zendio 用户支持与联系指引

公开访问版本：https://github.com/Lefeaker/AllinOB

最后更新：2025-02-14

面向 Zendio Chrome 扩展用户的支持与沟通流程说明。可将本页托管至官网或 GitHub Pages，供审核提交及用户快速定位帮助渠道。

## 1. 支持渠道

- **电子邮箱**：allinobsidian@outlook.com
- **GitHub Issues**：<https://github.com/aiiinob/all-in-obsidian/issues>，用于功能建议与缺陷反馈。
- **紧急通道**：遇到数据丢失或安全风险，可在邮件标题加注“URGENT”，我们会在 SLA 内优先处理。

## 2. 服务范围

- 安装、升级、卸载等基础问题。
- 剪藏、AI 解析、Obsidian 写入失败的排查。
- API Token、权限配置、网络连接等环境相关指导。
- 隐私、权限、数据安全类问题解释。

## 3. 响应与跟进

- **工作日响应时间**：48 小时内首轮回复；复杂问题提供预计解决时间。
- **问题登记**：收到请求后生成工单编号（格式 `AO-YYYYMMDD-###`），以邮件回复形式反馈给用户。
- **状态更新**：重大修复将在 GitHub Issue 与发布公告中同步；用户可通过原渠道查询进展。

## 4. 常见问题（FAQ）

### Q1：扩展无法连接 Obsidian，本地 REST API 报错怎么办？

- 确认 Obsidian 已启用 REST API 插件，并允许来自 `http://127.0.0.1` 或指定端口的访问。
- 在扩展设置中重新输入 Vault ID 与 API Token，保存后重试。
- 若仍失败，请附上控制台错误截图与 `chrome://extensions` 调试日志联系我们。

### Q2：剪藏内容丢失或覆盖原笔记怎么办？

- 检查模板设置是否开启“覆盖模式”。如需追加写入，请选择“追加到末尾”。
- 建议在 Obsidian 启用版本历史或使用 Git 以便快速回滚。
- 如需恢复，请提供笔记名称、操作时间，我们将指导您从备份中还原。

### Q3：是否会上传我的笔记或网页内容到云端？

- 本扩展仅在本地处理数据，不会向开发者服务器发送内容。详见《隐私政策》<../privacy-policy.md>。

### Q4：如何申请功能或提交 Bug？

- 在 GitHub Issues 选择对应模板（Feature Request / Bug Report），填写复现步骤、浏览器版本、扩展版本，我们会在排期评估后反馈结果。

## 5. 反馈流程示例

1. 用户通过邮箱或 Issue 提交问题，附带以下信息：
   - 浏览器版本与扩展版本号
   - 操作步骤、期望结果与实际结果
   - 相关截图、日志或导出的调试信息（如涉及隐私，请先脱敏）
2. 支持团队在 48 小时内确认收录并回复处理状态。
3. 如需进一步信息，将通过原渠道沟通；问题解决后同步结论与版本信息。
4. 对于重复出现的共性问题，会新增 FAQ 或发布修复公告。

## 6. 数据与隐私相关请求

- 访问、删除、限制处理或转移数据的诉求，可通过邮箱提交，我们将在 15 个工作日内完成验证与响应。
- 如需导出日志，请先审阅内容并删除敏感数据，再发送给我们。团队收到后仅用于排查当前问题。

## 7. 文档维护

- 每次版本发布前复核本页面，确保联系方式、SLA、FAQ 内容与实际运营一致。
- 建议在主仓库 `docs/` 目录及发布站点同步更新，并在 README 或扩展“帮助”入口指向本页。
